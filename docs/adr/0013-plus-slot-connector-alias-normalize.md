# `+` slot 連接符別名與粵拼錨 normalize 邊界

領域詞彙：見 [CONTEXT.md](../../CONTEXT.md) § 星號錨、粵拼錨。延續 [ADR-0012](0012-wildcard-code-anchor-query-syntax-v2.md)（通配碼錨／星號槽 v2）、[ADR-0004](0004-reference-reading-query-normalize-position-match.md)、[ADR-0005](0005-jyutping-anchor-query-syntax.md)。

Grill 共識：半形 `+`、全形 `＋` 與 `*` 輸入等價；canonical 仍 `*`；連續 slot 連接符唔支援；二字 `23o` 與三字 `23+o` 分軌；粵拼錨 normalize 範圍有限。

## 決定

1. **輸入別名**（`normalize_query_syntax` 或同層）  
   - 半形 `+`（U+002B）、全形 `＋`（U+FF0B）、既有 `＊`／`﹡` → 規範形 `*`。  
   - **唔**收 `﹢`、`⊕` 等其它 plus 字元。  
   - **唔**把 UI **查詢分頁** `+` 掣當查詢運算符。

2. **適用範圍（A+B+C+E）**  
   - 碼尾／中格／頭格**星號錨**（如 `23+就`、`2+好3`、`+門0`）。  
   - **通配碼錨**擴槽（如 `?30+人`）；非相鄰多連接符允許（如 `*香?30+人`）。  
   - 首格／中格 normalize 插入（如 `?+就=`、`?+你?`）。  
   - Legacy 碼尾 `&`／`·` 維持映射 `*`。  
   - **粵拼錨**缺字族（見下 §4）；**唔**對二字 `23o`、`3hon4` 硬插。

3. **slot 連接符語法**  
   - 右邊**必須**緊接漢字參考字或粵拼錨（拉丁）；**唔**可緊接 0243 碼（`2*好*3` ✗）。  
   - **只禁相鄰**多個連接符（normalize 後 `**`）；要多通配槽用 `?`（例 `?30?+人`，唔用 `?30++人`）。  
   - 非法 → **空結果 + hint**（連續連接符、`+` 後接碼各一句）。

4. **粵拼錨 normalize（10c 雙寫入）**  
   - `?yut?`／`?m?`／`?hon` → 通配左側插 `*`：`?*yut?`、`?*m?`、`?*hon`（`+` 等價）；舊寫法靜默收斂、唔 hint。  
   - `{首碼}?{音節}{末碼}`（如 `3?ngo4`）→ `?` **改寫**為 `*`／`+`（`3*ngo4` ≡ `3?ngo4`）。  
   - **二字韻母末格** `23o` ≡ `23我=`（**詞長 2**）；**唔**等同 `23+o`。  
   - **三字碼尾** `23+o`／`23*o` ≡ `23*我=`（**詞長 3**；結構 **碼｜碼｜韻錨**，同 `23*就=`；`o` 為韻母 shorthand）。  
   - **二字音節首格** `3hon4`（**詞長 2**）；**唔**改 `3*hon4`。  
   - **`23我=`（2 字）≠ `23*我=`（3 字）** — 不可合併 MatchSpec。

5. **Hint 文案（固定）**  
   - 連續：`唔支援連續 *／+：通配音節請用 ?，slot 連接符最多一個。例：?30?+人（唔好寫 ?30++人）。`  
   - `*` 後接碼：`*／+ 後須接漢字或粵拼錨，唔可以接碼。例：尾格用 2*好3 或 2*好人。`

## Considered Options

- **全域 blind `+`→`*`** 且 `23+o`≡`23o` — 混淆二字／三字；拒絕。  
- **2b 全串最多一個 `*`** — 打爆 `*香?30+人`；拒絕。  
- **2a 逐字 `++`→多槽** — 與 `?` 通配重疊；拒絕（2c）。  
- **10b 粵拼錨硬切換** — 創作者仍用 `?hon`；拒絕（10c）。  
- **合併 `23o` 與 `23+o`** — 詞長不同；拒絕。

## 分階段實作（建議）

| 階段 | 內容 | 可驗收 |
|------|------|--------|
| **P0** | `+`／`＋`→`*`；連續 `**` reject + hint；`*` 後接碼 hint | `23+就`≡`23*就`；`?30++人` hint |
| **P1** | 粵拼錨 normalize（`?*hon`、`3*ngo4`）；`23+o` 三字 parse | `?hon`≡`?*hon`；`23+o`≠`23o` 詞長 |
| **P2** | 指南／`check_guide_examples`；回歸 | CI green |

## Consequences

- `normalize_search_query` 增別名與非法形狀檢查（宜在 `+`→`*` 之後、`parse` 之前）。  
- 可能需要 **jyutping 碼尾 `23*o`** parser 分支（三字韻母錨，同 star tail 族）。  
- [ADR-0012](0012-wildcard-code-anchor-query-syntax-v2.md) 語意不變；本 ADR 只規範**輸入別名**與**normalize 邊界**。
