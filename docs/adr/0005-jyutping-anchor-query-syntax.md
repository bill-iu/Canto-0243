# 粵拼錨與三格韻錨查詢語法

領域詞彙：見 [CONTEXT.md](../../CONTEXT.md) § 粵拼錨、完整音節粵拼錨、三格韻錨查詢、韻母粵拼錨、聲母粵拼錨。延續 [ADR-0002](0002-mask-family-dispatch-spec-boundary.md)。

創作者希望以拉丁字母在**缺字型查詢家族**內取代**參考字**（碼字、韻／聲錨、碼夾等號、三格查詢），同時修復 `?港=?` 等三格漢字語法誤分派。我們決定：

1. **三格韻錨查詢** — 漢字側 `?{字}=?`（如 `?港=?`）為三字、**中格韻母**錨；解析優先於含 `=` 的 mask 誤判與 WordLookup。
2. **粵拼錨三子類** — 聲母（單輔音）、韻母（`a/e/i/o/u/m`、獨立 `ng` 與 `m` 等價、片段如 `yut`/`ei`）、完整音節（聲韻齊、唔限聲調，如 `hon`→韓／漢／寒／汗）。
3. **碼音節查詢** — 二字 `3hon4`（首格完整音節）；三字 `{首碼}?{音節}{末碼}`（如 `3?hon4`、`2?ngo4`），首碼後 `?` 表示多一音節槽；二字末格 `?hon`；碼後拉丁 `{碼}{音節}`／`{碼}{韻母}`（如 `23ngo`、`23o`）錨**末格**。
4. **分派優先序** — 缺字家族（含粵拼錨模板）優先於整段**粵拼查詢**；解析唔到收錄讀音時拒絕並提示，唔降級返空。
5. **範圍** — 僅 **0243搜尋模式** 缺字家族；**近反義關係查詢**唔收粵拼錨。

**Considered Options**

- 近反義支援 `33~ngo` — 同音節多字種子歧義大，排除。
- `23?ngo4` 三字左碼 — 改為 `{首碼}?{音節}{末碼}` 各一位碼（D1）。
- 全部等價 rewrite 成漢字再 parse — 完整音節錨需同音節 union，唔應收窄為單一參考字。
- `?yut=` 三格 — 創作者改為 `?yut?`（韻母片段、省略 `=`）。

**Consequences**

- `query_parse` 增三格韻錨與粵拼錨正規化；`SlotConstraint` 擴充粵拼錨 kind 或等價 phoneme 解析。
- `?港?`（中格字面）與 `?港=?`（中格韻母）語意分離；`?yut?` 與 `?syut?`（完整音節中格）分離。
- 現有 `parse_rhyme_anchor_query` 二字尾錨（`?就=`）不變。
