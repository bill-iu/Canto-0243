# 單檔案程式碼不超過 300 行

以 Ponytail 精神（能一行就不寫十行、刪減優於新增）為指導，我們決議：任何單一檔案的程式碼行數不得超過 300 行（不含空行與註解）。此限制旨在強制模組深層化（depth），避免淺層（shallow）模組Expand，並提升局部性（locality）與可測試性。

**Considered Options**

- 200 行上限 — 過於嚴格，多數現有檔案（如 `query_dispatch.py`）將需強制拆分，破壞原有邏輯聚合度。
- 500 行上限 — 過於寬鬆，無法有效防止程式碼膨脹與複雜度累積。
- 300 行上限 — 平衡點：足夠容納完整功能模組（如一個 query parser 或一個 repository），又能強制過大的類/函數進一步拆分。

**Consequences**

- 現有超過 300 行的檔案（如部分 `services/*` 檔）需逐步重構或拆分。
- 新功能開發時，如預期超過 300 行，須提前設計模組拆分方案。
- 300 行為**軟限制**：特殊情況（如 auto-generated code、測試 fixture）可豁免，但需在 PR 中明確標註 `ponytail: 300-line limit exemption — {reason}`。
- **測試檔案**不在此限（測試邏輯常需多個 scenarios）。

**非目標**

- 此限制**不是**品質的充分保證（300 行淺層程式碼仍然壞）。
- 此限制**不是**硬規則（1 行不合規不會阻擋 PR，但連續違反將在 code review 中被挑戰）。
