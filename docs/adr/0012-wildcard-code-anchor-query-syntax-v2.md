# 通配碼錨與星號槽查詢語法 v2

領域詞彙：見 [CONTEXT.md](../../CONTEXT.md) § 通配碼錨查詢、單字韻錨查詢、碼＋參考字＋通配（中格韻）、星號錨、缺字查詢。延續 [ADR-0002](0002-mask-family-dispatch-spec-boundary.md)、[ADR-0004](0004-reference-reading-query-normalize-position-match.md)、[ADR-0005](0005-jyutping-anchor-query-syntax.md)（**部份 supersede**，見下）。

Grill 共識：`?30人` 等查詢唔應再當「逐字元四格」缺字；`*` 統一標音節槽／頭格錨；首格字面可 normalize；碼左參考字必須用 `=` 標韻母。**硬切換**，唔保留舊四字 `?30人` 別名。

## 決定

1. **Normalize 層**（`normalize_search_query` 或同層）  
   - 首格字面：`X…` → `*X…`（與 `*X…` 同一 MatchSpec）。  
   - 中格字面／韻錨：僅漢字**左邊係通配符**時 `?{字}…` → `?*{字}…`（如 `?你?`、`?你=?`）。  
   - **唔**對 `?30人`、碼左參考字、`23?` 等插 `*`。

2. **通配碼錨查詢** `{通配}{連續碼}{參考字}`  
   - 詞長 = 1 + 碼位數；碼約束**第 2…N 音節**；尾參考字佔最後音節槽、同韻（無 `=`）。  
   - 例：`?30人` → 三字（「讀書人」`230` 中；「中年人」`300` 唔中）。  
   - 四字：`?30*人`（`*` 多一音節槽，見下）。  
   - 中間夾 `?`：左至右掃描（`?3?0人`、`?30?人`）。  
   - **硬切換**廢止舊 MaskQuery 四字 `?30人`。

3. **單字韻錨** `?{參考字}=`（**無** `*`）  
   - 詞長 1；0243 任意；同參考字韻母。  
   - **與** `?*就=`（二字）**完全分開**；廢止舊二字 RhymeAnchor `?就=`（「之後」類）。

4. **星號槽**  
   - 每個 `*`（非句首頭格連接符）→ 相對無該 `*` **多一個音節槽**。  
   - 句首 `*{字面}…`：頭格連接符，唔多音節（`*門0` ≡ `門0`）。  
   - `?*{參考字}=` → 二字（如 `?*就=`）。

5. **碼＋參考字＋通配（中格韻）** `?{碼}{參考字}=?`  
   - 三音節；中格碼 + 同參考字韻；必須 `=`。  
   - `?{碼}{參考字}?` 無 `=` → 碼位同字面矛盾 → **空結果 + 一行 hint**（提示 `?3人=?`）。

6. **純碼＋通配**（`23?`、`3_`）  
   - 維持逐字元缺字，唔加 `*`（[CONTEXT](../../CONTEXT.md) 明確排除）。

7. **解析 → MatchSpec**  
   - 新族在 `query_parse` 分類；`normalize_to_match_spec` 產出 MatchSpec；`execute_match_spec` 執行。  
   - 矛盾串可 `UnmatchedQuery` 或 SearchResult hint，唔入 DB。

## Considered Options

- **保留舊 `?30人` 四字** — 與創作者意圖及「碼指音節序」矛盾；拒絕。  
- **`?就=` normalize 成 `?*就=`** — 單字 vs 二字語意不同；拒絕。  
- **`?3人?` 逐字元四字** — 碼左「人」應為參考字；改為必須 `?3人=?` 或 hint。  
- **全部缺字強制 `*`**（含 `23?`）— 易與 `23*就` 混淆；拒絕。  
- **靜默空結果**（矛盾串）— 創作者選 **hint**（B）。

## Supersedes / 衝突

- [ADR-0005](0005-jyutping-anchor-query-syntax.md) 結尾「`?就=` 不變」→ **作廢**；改為單字韻錨 + `?*就=` 二字。  
- 舊指南「缺字每字元一格」→ 仍適用**純逐字元**與**碼＋通配**，唔適用通配碼錨族。

## 分階段實作（建議順序）

| 階段 | 內容 | 可驗收 |
|------|------|--------|
| **P0** | Normalize rewrite（首格 `*X`、通配左中格 `?*字`）；`*香??` 頭格 + 尾 mask 解析 | `門0` ≡ `*門0`；`香??` ≡ `*香??` |
| **P1** | 通配碼錨 parser + MatchSpec + 執行（`?30人`、`?30*人`、掃描 `?3?0人`） | 「讀書人」中、「西班牙人」唔中（三字） |
| **P2** | 單字韻錨 `?就=`；`?*就=` 二字；廢止舊 RhymeAnchor `?就=` | 單字同韻；唔返「之後」 |
| **P3** | `?3人=?` 三格；`?3人?` hint | hint 文案；中格同韻 |
| **P4** | `*香?30人` 頭格 + 通配碼錨餘段；前端指南更新 | 四音节用例 |
| **P5** | 刪舊 MaskQuery 路徑對已覆蓋串的誤判；回歸測試／`check_guide_examples` | CI green |

每階段：**先測試後實作**（TDD）；硬切換唔留舊四字 `?30人` 雙軌。

## Consequences

- `word_query_parser.py`、`query_parse.py` 增分類與 normalize；`looks_like_mask_query` 須排除通配碼錨形狀。  
- `frontend/index.html` 指南：缺字／星號／韻錨示例改用 `*` 規範寫法；標註 `?30人` 新語意。  
- 維護者查「為何 `?30人` 唔再出西班牙人」→ 本 ADR + CONTEXT § 通配碼錨。  
- 粵拼錨 `?yut?` 等 **本 ADR 唔改**（P0–P5 外，除非 parse 衝突）。
