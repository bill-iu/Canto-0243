# Canto-0243

離線粵語填詞查韻工具：依 0243／02493 數字碼、粵拼與韻母規則，幫創作者找出可替換的詞條與近義／反義關係。

## 核心資料

**詞條**：
字典中一條可查記錄，由字面、0243 碼與粵拼組成；同一字面可有多條詞條（多讀音）。
_Avoid_：Word（實作名）、候選（僅在描述搜尋結果清單時使用）

**字面**：
詞條上顯示的漢字序列（一至多字），例如「香港」的字面長度為 2。
_Avoid_：詞面、字串、char（實作名）

**0243 碼**：
附在詞條上的數字序列；每一格對應一個音節的聲調，經固定映射轉成 0/2/3/4/9。長度與字面音節數一致（「香港」→ 兩位碼）。
_Avoid_：code（實作名）、音碼、把 0243 碼當韻母鍵盤映射

**粵拼**：
詞條的粵語羅馬字標注，一音節一段（如「香港」→ `hoeng1 gong2`），與 0243 碼逐音節對應。
_Avoid_：jyutping、拼音（易與普通話混淆）、拉丁、拉丁字母、字母（作產品或教學文案時）

**讀音**：
詞條上完整音節的粵拼（如「你」→ `nei5`）。查詢比對時，參考字多半只提供**聲母或韻母其中一維**作錨；0243 碼數字則單獨約束該格聲調，與參考字是 `nei5` 還是 `nei1` 無關。
_Avoid_：把參考字的整段讀音等同於該格的查詢條件

**聲母**：
粵拼音節的輔音部分（如 `gong2` 的 `g`）；同聲比對以聲母為單位。
_Avoid_：initial、字頭

**韻母**：
粵拼音節的韻部（含調，如 `gong2` 的 `ong2`）；同韻比對以韻母為單位，與 0243 碼的聲調維度不同。
`j-` 後接 `y-` 的音節（如 `jyut`、`jyu`、`jyun`）：聲母為 `j`，`y` 屬韻母一部分（`yut`、`yu`、`yun`），不拆成聲母叢集 `jy`。
**獨立鼻音韻母**：粵拼**整個音節**僅為 `m` 或 `ng`（加調，如唔 `m4`、吾 `m4`、五 `ng5`、午 `ng5`）；該字面**無聲母**，查詢比對上**只**屬韻母維度；`m` 與 `ng` **完全等價**（mrpinyin M／NG 欄）。唔同「門」`mun` 嘅輔音聲母 `m`，亦唔同「我」`ngo` 嘅聲母叢集 `ng`。
_Avoid_：final、字尾、把唔／五／午等獨立鼻音音節當有聲母、把 codec 存入 `initials` 嘅 `m`／`ng` 當聲母錨匹配

## 比對關係

**同韻**：
兩個字面在對應音節位置上韻母一致；常以 `=` 或參考字標示（如 `?就=`、`香港=`）。
_Avoid_：押韻（指比對規則時）、rhyme

**同聲**：
兩個字面在對應音節位置上聲母一致；常以 `=` 與參考字標示（如 `?=就`、`=香?`）。
_Avoid_：用「同音」泛指同韻或同聲、initial（實作名）

**同音**：
在本工具中，指對應音節上的 0243 碼與查詢數字一致（0243模式下含等價變體）；不指整個音節聽感完全相同。
_Avoid_：用「同音」泛指同韻或同聲；同碼（若 UI 已用「同音」則不必並列）

## 搜尋模式

**0243搜尋模式**：
以 0243／02493 數字碼為核心的搜尋，含 **0243模式** 與 **02493模式** 兩檔；與 **近反義模式** 並列為頂欄三大 pill 中之兩檔。
_Avoid_：鬆／緊並稱作模式正名、把 0243搜尋模式當成近反義的同義詞

**0243模式**：
0243搜尋模式預設檔：0243 碼比對時允許等價變體。介面選單旁註顯示「鬆」。
_Avoid_：鬆模式（作領域正名）、m1（實作名，僅程式與 URL）

**02493模式**：
0243搜尋模式進階檔：在 0243 基礎上**分清二聲**（9 鍵聲調區分）。介面選單旁註顯示「緊」。
_Avoid_：緊模式（作領域正名）、m2（實作名，僅程式與 URL）

**近反義模式**：
以詞義關係為主的搜尋，回傳近義、反義與語意相關詞條。介面 pill 可簡寫「近反義」；旁註顯示「查」。教學與文件用全名。輸入**近反義關係查詢**語法時觸發**搜尋模式轉接**（見下），唔再空結果。
_Avoid_：syn、查模式（作為模式正名）、關係模式

**近反義前次0243搜尋模式**：
用戶切換至**近反義模式**之前所選的 **0243搜尋模式** 檔（**0243模式**或 **02493模式**）；**搜尋模式轉接**時還原此檔執行**近反義關係查詢**。若無紀錄（如首次進站即在近反義），預設 **0243模式**。**API 保險轉接**時可由客戶端以 query param `fallback_0243_mode` 傳遞（優先）；同名 HTTP header 可選；皆缺則預設 **0243模式**。
_Avoid_：lastMode、m1/m2（實作名）

**搜尋模式轉接**：
在**近反義模式**下，若查詢串屬**近反義關係查詢**語法，自動切換至**近反義前次0243搜尋模式**（無則 **0243模式**），更新頂欄模式與 URL，並以該檔執行查詢、展示結果；唔再回傳空列。純漢字池查詢（如「開心」）仍留喺近反義模式。轉接時**重設分頁**（`offset=0`）。轉接時結果區顯示一行**轉接提示**（說明已切換至哪個 0243 檔）。**近反義前次0243搜尋模式**僅當次頁面生命週期有效。執行分兩層：**介面轉接**（送出請求前先切模式與 URL）與 **API 保險轉接**（仍收到 `mode=近反義` 且為關係語法時，後端改以客戶端提供或預設的 0243 檔執行並回傳**轉接提示**與 **`X-Effective-Mode`**；Web UI 讀到時同步校正 pill 與 URL）。
_Avoid_：模式偷換（UI 仍顯示近反義）、把缺字／碼字查詢一併轉接、僅後端轉接而介面唔更新、轉接後沿用舊 offset

**轉接提示**：
**搜尋模式轉接**發生時，於結果區頂部顯示嘅一行說明（如「此語法已切換至 0243模式（鬆）查詢」）；與查無結果、粵拼在近反義模式被拒等既有 `hint` 同列，唔阻塞結果展示。**介面轉接**由前端本地生成；**API 保險轉接**由後端 `hint` 回傳；兩層文案一致。
_Avoid_：toast、彈窗確認

**語意相關**：
與查詢詞在語意上有關聯、但非近義亦非反義的詞條（近反義模式第三類結果）。
_Avoid_：相關詞（不區分類型時）、semantic_related

**近反義池**：
對一個字面，合併 `word_relations` 與**靜態詞林**後、經排序與去重的近義、反義與語意相關候選集合；候選必須是**有效字面**（含漢字、無詞林編碼或英數噪音）；**近反義模式**與**近義橋反義** ingest 共用同一建池規則。
_Avoid_：relation pool（作領域正名）、syn_ant_service（作池名稱）

**近反義池投影**：
Runtime 讀取**近反義池**的統一入口；近反義模式、`~` / `!` 查詢與關係補錄擴展互斥皆經此投影取得池快照或字面集合，建池規則本身不變。
_Avoid_：pool facade、runtime 直呼 build_pool

**靜態詞林埠**：
bundled 近義／反義語料（詞林 cilin、國語辭典近義檔、反義語料檔）的 **raw lookup** 邊界。Runtime `~`／`!` 以合併近義、反義 getter 消費；ingest 可用分源 getter（cilin、guotong、antisem）與種子字面全集枚舉，寫入 `word_relations` 時保留來源標籤。資料載入與模組內 dict 為實作細節，對外只經埠存取。
_Avoid_：ThesaurusPort（作領域正名）、static thesaurus（作中文正名）、直接讀模組級 dict

**收錄決策**：
對一個字面，在 raw lookup 之上判定能否注入詞條庫、讀音來自何處（單字 rime、多字詞級標音、或音節拼接讀音），輸出三分類結果。**參考字讀音解析**須經此決策鏈；`ensure_word_in_db` 僅依決策結果執行 DB 注入，不重複收錄規則。
_Avoid_：admission（作領域正名）、在 ensure 層重複拼接讀音邏輯、把收錄判斷藏進詞庫埠 lookup、快取優先於收錄決策當讀音權威

**參考字讀音解析**：
依**收錄決策**解析**參考字**讀音，對外兩條入口：（1）**錨點音素選項**——單字**參考字**在**韻／聲錨**所在音節的聲母或韻母選項集合（如「香」在 `香=?` 取韻母 `oeng`，在 `=香?` 取聲母 `h`）；多讀音時 **union 全部可用選項**（如「行」同時含 `hang` 系韻部與 `hong` 系韻部）。（2）**等號參考讀音**——**等號查詢**與**碼夾等號查詢**所用整詞字面的粵拼音節序列（聲母列或韻母列）；多讀音時取 **pron_rank** 權威單列，**不** OR 比對（與**碼夾等號查詢**約束一致）。`pron_rank` 平手時依 **Essay 詞頻**（字面級）降序，再平手略過 `aa` 變體寫法（如 `haang` 視同 `hang` 族）後依粵拼字串升序選單列。呼叫端明示是否允許注入詞條庫——**查詢分派**與**缺字型查詢執行**的比對路徑可注入，**詞庫快取索引**取向量候選與預載路徑不注入。注入成功後才更新**詞庫快取索引**；解析本身不隱式 sync 既有詞條。不以快取為讀音權威。
_Avoid_：phoneme_lookup（作領域正名）、在索引候選解析時隱式 ensure 或 sync、把參考字整段讀音當比對條件、在 match_equals 重複 ensure 編排、等號路徑對多讀音做 OR

**關係寫入**：
將近義、反義列寫入詞條關係表的領域操作；含 canonical 字面對排序、收錄字面索引、去重插入。Runtime 與 ingest 共用同一寫入規則；ingest 僅作批次 adapter。
_Avoid_：merge pipeline（作領域正名）、把 ingest 套件當 runtime 依賴

**關係圖**：
對已收錄字面，查詢**直接近義鄰居**（`word_relations` 雙向近義 **＋** 靜態詞林近義）與**直接反義鏡射對**（`!` expand 同源）。近反義池建構、創作者手動關係擴展、近義橋反義 ingest 共用同一鄰居規則。
_Avoid_：CharRelationGraph（作領域正名）、在 ingest 繞過靜態詞林埠直讀模組 dict

## 查詢語法

**查詢分派**：
將創作者輸入分類為等號、碼字、缺字等語法類型，正規化為可比對規格並路由至對應搜尋路徑；**分派優先序**本身是領域規則的一部分。執行層（如**缺字型查詢執行**）不應重複語法解析或 ParsedQuery→規格轉換。
_Avoid_：parser、handler chain（實作細節）、在 position_match 各處散落 build_* 正規化

**缺字型查詢執行**：
接收**查詢分派**正規化後的比對規格，將缺字查詢、韻／聲錨、碼字查詢、字面參考、碼夾等號查詢，以及標有 `compound_kind` 的近義／反義複合查詢，統一執行為搜尋結果；不應再依 ParsedQuery 型別分支或重複語法正規化。語法上 ~~／!! 仍不屬缺字型查詢家族。
_Avoid_：位置查詢、position match、候選來源（作領域正名）、在執行層保留 isinstance 分派梯

**近反義關係查詢**：
在 **0243搜尋模式**（0243模式或 02493模式）下，以 `~`（近義）或 `!`（反義）查詞義關係；可帶前綴 0243 碼（如 `33!開心`）。`!!` 表示**反義複合詞**查詢；`~~` 表示**近義複合詞**查詢；`!{連接}!`／`~{連接}~` 表示**連接詞複合查詢**。於**近反義模式**輸入時觸發**搜尋模式轉接**。全形 `～`／`！` 與半形 `~`／`!` 等價；缺字 wildcard 的 `？` 與 `?` 等價——**查詢分派**入口正規化為半形後再解析。
_Avoid_：稱此為「開啟近反義模式」（那是近反義模式的另一條路徑）

**反義複合詞**：
填詞常用的兩字對舉詞（如「生死」「是非」）；`!!` 只回傳詞庫存在的 2 字詞條。語法與 `~~` 對稱：`!!`、`33!!`、`!!你`、`33!!你`；執行於 **0243搜尋模式**（**近反義模式**輸入時經**搜尋模式轉接**）。候選來自：（1）專案 **curated 反義複合列表** ∩ 詞庫，**順序對稱**（AB 與 BA 視為同一候選，如 `你我`／`我你`）；（2）詞庫內**反義素複合**掃描（兩字互為直接反義鄰居）；（3）**單字反義合成**（每字 top‑K=12 反義鄰居組合 AB／BA，再對詞庫）。三者 union 去重。啟動時預算（1）（2）字面快取；（3）查詢時合成。結果排序：curated → 反義素掃描 → 合成，組內用搜尋結果排序 tier。**唔**查詢時造字或拼接收錄缺字面。候選由 **反義複合** domain 模組對外提供（與 **近義複合** 對稱）。
_Avoid_：一般雙字詞、僅因資料管道產生的單字反義 pair、只匹配 curated 正寫順序、把 `!!` 當成僅 curated 列表

**反義複合快照**：
**離線啟動預載**就緒後、供 `!!` 查詢讀取的候選字面集合，涵蓋 curated ∩ 詞庫（含順序對稱）與反義素掃描兩源；**單字反義合成**不在快照內，於每次查詢時追加 union。快照由單一反義複合模組對外提供，預載編排與 executor glue 不各自定義候選規則。
_Avoid_：compound_ant_cache（作領域正名）、把查詢時合成結果寫回快照

**填詞連接詞**：
**連接詞複合查詢**中格允許嘅單字連接詞，固定集合（11 字）：**與、和、或、共、同、及、跟、而、且、並、向**。中格字面須與查詢串完全一致。**唔**收「之、以、為、若、如、比」等——多為「之於」「以為」或比喻結構，唔係填詞并列對舉。
_Avoid_：把任意虛詞都當填詞連接詞、把連接詞當通配符、為湊字面數盲目擴充集合

**連接詞複合查詢**：
三字詞查詢；中格為**填詞連接詞**，首尾兩字須互為反義（`!{連接}!`，如 `!與!`、`349!與!`）或近義（`~{連接}~`）。可帶尾韻錨字（如 `~與~你`）。只返詞庫已有嘅三字詞；首尾關係判定沿用**反義複合詞**／**近義複合詞**候選規則（含掃描與合成）。**順序對稱**：`A{連接}B` 與 `B{連接}A` 視為同一候選（如「生與死」／「死與生」），仍須詞庫有該字面。**唔**查詢時造字。執行屬缺字型查詢家族；語法上分派獨立於二字 `!!`／`~~`。
_Avoid_：把 `!與!` 當整詞 lookup、只掃中格連接詞而唔查首尾關係、把連接詞複合併入一般 `?` 遮罩而無關係約束

**近義複合詞**：
填詞常用的二字近義複合（如「恐懼」「美麗」），兩字義素在字級近義關係上可視為同義或近義；`~~` 只回傳詞庫存在的 2 字詞條。語法與 `!!` 對稱：`~~`、`33~~`、`~~你`、`33~~你`；執行於 **0243搜尋模式**（**近反義模式**輸入時經**搜尋模式轉接**）。候選來自：（1）專案 **curated 近義複合列表** ∩ 詞庫；（2）詞庫內**同義素複合**掃描（兩字互為直接近義鄰居、無 direct 反義）；（3）**單字近義合成**（每字 top‑K=12 近義鄰居組合，再對詞庫）。三者 union 去重。啟動時預算（1）（2）字面快取；（3）查詢時合成。結果排序：curated → 同義素掃描 → 合成，組內用搜尋結果排序 tier。第一版僅 runtime，不 ingest 寫入關係表。第一版不自動猜測義素互補複合（可人工收進 curated 列表）。
_Avoid_：把一般雙字並列詞都當近義複合、用「同義複合」泛指所有 2 字詞

**近義複合快照**：
**離線啟動預載**就緒後、供 `~~` 查詢讀取的候選字面集合，涵蓋 curated ∩ 詞庫與同義素掃描兩源；**單字近義合成**不在快照內，於每次查詢時追加 union。快照由單一近義複合模組對外提供，預載編排與 executor glue 不各自定義候選規則。
_Avoid_：compound_syn_cache（作領域正名）、把查詢時合成結果寫回快照

**源 3 tier 快取**：
同一 server process 內，首次 `~~` 查詢時 lazy 預算**單字近義合成**的 tier 對照表並重用，以加速後續 `~~`／`~~港` 查詢；屬查詢效能優化，不擴張**近義複合快照**字面集合，不重啟即失效。
_Avoid_：把源 3 tier 快取當快照、跨請求 LRU（第一版不做）、ingest 寫回

**近義橋反義**（ingest 補全）：
對**有近義、無反義**的字面：在其近義候選中（與 runtime `~` 相同來源：`word_relations` 近義 **＋** 靜態詞林），以語意向量找最相似、且本身已有反義的字面作**橋接近義**（候選內 cosine argmax，無額外門檻），將該橋接詞的**全部直接反義**（同 `!橋接詞`、不 expand）借作原字的反義，寫入關係表（`source=ant_syn_bridge`，`score`=W–S cosine）並去重。僅 ingest 階段執行，不於 runtime 載入 embedding。
_Avoid_：反義橋接（作泛稱）、runtime 即時猜反義

**創作者手動關係**：
創作者於**關係補錄**查詢分頁指定**種子字面**與**對端字面**（兩者須已收錄於詞條庫），選擇近義或反義；後端寫入一條 direct `word_relations`（`source=manual`）。若該 direct 關係（同字面對、同關係類型）已存在則整筆拒絕並提示。寫入成功後依類型一跳擴展：`manual` 近義 → 種子字面接上對端字面在 DB 與靜態詞林的**直接近義鄰居**，衍生 `source=manual_syn_cluster`；`manual` 反義 → 種子字面接上對端字面直接近義鄰居作反義（同 runtime `!` 鏡射），衍生 `source=manual_ant_mirror`。**擴展互斥**：一跳衍生不得寫入種子字面已具相反關係類型的字面（近義擴展跳過既有反義鄰居；反義鏡射跳過既有近義鄰居），避免同一字面同時出現在 `~` 與 `!` 結果。衍生列與 direct 去重；**direct 列排序信號優先於衍生列**。提交結果以筆數摘要回饋創作者。不佔查韻首屏主體。
_Avoid_：手動關係、使用者新增關係（作領域正名）、把創作者手動關係稱作近義橋反義

**撤回創作者手動關係**：
在**關係補錄**以與補錄相同的種子字面、對端字面與關係類型，刪除對應的 direct `source=manual` 列，以及當初補錄時可重算的一跳衍生列（`manual_syn_cluster` 或 `manual_ant_mirror`）。僅限創作者手動寫入；`cilin`、`ant_syn_bridge` 等 ingest 資料不受影響。找不到 direct manual 列則拒絕並提示。
_Avoid_：改類型、undo（作領域正名）、撤回 ingest 關係

**關係補錄**：
創作者手動關係功能的專用 view；以**查詢分頁**承載於**查韻介面**（`index.html`），頂欄可開新分頁進入；不再提供獨立的 `relation-entry.html` 頁面。
_Avoid_：新增關係頁、relation form（作產品名稱）、把關係補錄當成必須另開瀏覽器分頁的獨立站點、保留雙軌的關係補錄 HTML

**關係補錄命令**：
創作者於**關係補錄**提交的補錄或撤回；兩者共用同一組字面驗證與一跳擴展計畫，再分別寫入或刪除 `word_relations`。
_Avoid_：manual command、關係寫入（作泛稱）

**參考字**：
查詢語法中標定某音節位置的漢字，用於同韻、同聲或字面約束（如 `23就` 的「就」、`2=我3` 的「我」）。
_Avoid_：錨字、錨點字

**粵拼錨**：
在**缺字型查詢家族**內，以粵拼取代**參考字**標示讀音約束；與**粵拼查詢**（整段粵拼查詞）無關。分三子類，由粵拼片段形狀與語法位置自動判斷：（1）**聲母粵拼錨**—單輔音（如 `3h4`↔`3=香4`；`m` 亦可作聲母，如「門」`mun`）；（2）**韻母粵拼錨**—單母音 `a`/`e`/`i`/`o`/`u`/`m`，或獨立韻母 `ng`（唔係音節 `ngo` 嘅聲母部分），或唔構成完整音節嘅韻母片段；韻母狀態嘅 `m` 與獨立 `ng` **完全等價**（如吾／唔／五／午等）；如 `23o`↔`23我`、`?yut?`；（3）**完整音節粵拼錨**—見下。**二字碼＋粵拼**（`23o`、`3hon4`）：粵拼片段作用似**參考字**／讀音約束。**韻母末格二字**（`23o`）：**詞長 2**；語意 **`23我=`**（整詞 code `23`、末格同「我」韻母 `o`）。**碼尾 slot 連接符 + 韻母／參考字三字**（`23+o`／`23*o`）：**詞長 3**；結構 **碼｜碼｜韻錨**（同 `23*就=`，`*` 係碼同尾錨連接符）；語意 **`23*我=`**（第 1／2 音節碼 `2`／`3`、第 3 格同「我」韻；`o` 係韻母粵拼 shorthand）。**唔**等同二字 `23o`／`23我=`。**完整音節首格二字**（`3hon4`）：**詞長 2**；code `34`、首音節粵拼同 `hon`；**唔**改寫為 `3*hon4`。**`+`／`*` 別名只適用**下列缺字型粵拼錨（同 P0 中格 normalize 對稱）：`?yut?`／`?m?`／`?hon` → 通配左側喺**粵拼錨前**插 `*`（`?*yut?`、`?*m?`、`?*hon`；`+`／`＋` 等價；**過渡期**舊寫法 `?hon` 等 normalize 收斂、唔 hint）；`3?ngo4` 類 **`{首碼}?{音節}{末碼}`** → **`?` 改寫為 `*`／`+`**（`3*ngo4`／`3+ngo4` ≡ `3?ngo4`；過渡期 `3?ngo4` 仍收）。教學只教規範形 `?*hon` 等。**slot 連接符**共用**星號錨**規則：右邊必須係漢字或粵拼拉丁錨（唔係數字碼）；只禁相鄰連續；連續 → hint。**近反義關係查詢**唔收粵拼錨。若片段對應唔到任何收錄讀音，**拒絕查詢並提示**。
_Avoid_：把 `23o`（二字）同 `23+o`／`23*o`（三字）合併、把 `23我=`（二字）同 `23*我=`（三字）合併、把 `3+hon4` 當 `3hon4`、混淆二字韻母粵拼錨同三格 `?yut?`
**歧義片段**：`m` 同 `ng` 喺粵拼入面可以係聲母定韻母；查詢語法須另有規則分派，**唔**可假設單一粵拼片段必為聲母，亦**唔**可假設必為韻母。
**碼夾單片段查詢**（如 `3m4`、`3h4`）：`{首碼}{單一粵拼片段}{末碼}` 預設**聲母粵拼錨**（含 `m`，如「門」`mun`）；**唔**表示韻母狀態嘅 `m`。韻母狀態嘅 `m` 改用三格 `?m?`／`?ng?`、韻／聲錨（如 `?唔=`）、或 `{碼}{韻母}` 末格（如 `23o`）。
**雙聲母粵拼錨**：粵拼聲母叢集 `ng`、`gw`、`kw`（如「我」`ngo`、`關` `gwaan`、`困` `kwan`）可寫 `{首碼}{叢集}{末碼}`（如 `2ng0`、`3gw4`），語意同 `{碼}{單輔音}{碼}` 聲母錨。`gw`／`kw` 僅作聲母；`ng` 與韻母狀態嘅獨立 `ng` 仍歧義，須另定分派（見下）。
**歧義粵拼錨雙列結果**：當同一查詢串可合理解讀為**聲母粵拼錨**或**韻母粵拼錨**（如 `{碼}m{碼}`、`{左}ng{右}`、三格 `?m?`／`?ng?` 中格）時，**唔**強制二選一；一次查詢同時執行兩維，結果像**近反義關係查詢**般分開兩節展示——**聲母列**（該格有**輔音聲母** `m` 或聲母叢集 `ng`，如「門」「我」；**唔**包含**獨立鼻音韻母**音節，含唔／吾／五／午等）同**韻母列**（該格**獨立鼻音韻母** `m`／`ng` 等價）。兩列獨立列出；同一詞條通常只出現喺一列。觸發清單：`m` 或 `ng` 作碼夾或三格中格錨定串。無歧義者（如 `3h4` 僅聲母、`23o` 僅韻母、`3gw4` 僅聲母、`?yut?` 鎖韻母片段）維持單列。分頁：`limit`／`offset` 對**聲母＋韻母合併序列**生效（同近反義模式）；每列項帶**錨維度**標籤供前端分節。
_Avoid_：把 `ngo` 嘅 `ng` 當獨立韻母 `ng`、假設單一粵拼片段必為聲母、假設必為韻母、把 `23ngo` 當 `23我` 韻母錨、拉丁錨（作產品名）

**完整音節粵拼錨**：
**粵拼錨**子類：該格粵拼（無調）與錨定串完全一致，聲韻同時滿足，候選為同音節字面（唔限聲調）。**二字碼音節查詢**：`3hon4`—碼 `34`、**首格**完整音節 `hon`（音節緊接首碼、中間無 `?`）。**三字碼音節查詢**：`{首碼}?{音節}{末碼}`（如 `3?hon4`、`2?ngo4`）—首碼與末碼各一位 0243 碼，中格完整音節；首碼後的 `?` 表示多一音節槽。**二字末格**：`?hon`（X韓類）；`{碼}{音節}`／`{碼}{韻母}`（如 `23ngo`、`23o`）—碼後粵拼錨**末格**（唔係首格）。**三字中格**：`?syut?`（完整音節）或 `?yut?`（韻母片段）。例：`23ngo`（末格完整音節 `ngo`）與 `23o`（末格韻母 `o`）語意唔同。
_Avoid_：把 `3?hon4` 與 `3hon4` 混為同一詞長、用 `23?ngo4` 當三字碼音節查詢、二字末格錨用 `?hon?`

**字面參考**：
以 `@` 連接參考字，要求該位置字面與參考字完全相同、不做同韻擴展（如 `23@就`）。
_Avoid_：字面錨、@錨

**通配符**：
查詢語法中 `?`、`_`、`%` 三者等價，表示該格**字面不限**（咩字都得）。用於**缺字查詢**各格，以及**三格韻錨查詢**／**粵拼錨**的前後格（如 `?yut?` 首尾兩格）；與中格讀音約束（參考字或粵拼錨）並存時，通配符只約束字面、不約束讀音。
_Avoid_：把中格粵拼或漢字錨當通配符、說通配符本身決定讀音

**缺字查詢**：
以 0243 碼、**通配符**或漢字描述字面形狀。**純逐字元**缺字仍係查詢串每字元佔一格、串長即詞長。**首格字面**漢字：`X…` 與 `*X…` **語意等價**；規範寫法 `*`，parser **normalize → 同一 MatchSpec**（過渡期雙寫入）。**中格字面**：僅當**漢字左邊係通配符**（`?`／`_`／`%`）時，`?{字}?` 與 `?*{字}?` **等價**並 normalize（如 `?你?` → `?*你?`）；**漢字左邊係 0243 碼**時唔套用（該漢字改走**通配碼錨**／參考字規則，如 `?30人` 尾「人」係參考字、唔佔額外字面格）。**僅碼＋通配**（`23?`、`3_`）：維持現寫法，唔加 `*`。通配碼錨、星號錨擴槽走各自族。
_Avoid_：位置查詢、mask 查詢、對 `?30人` 喺「人」前插 `*`、把 `*門=0` 同 `*門0` 當等價、首格 `*` 當多一音節、對 `23?` 硬加 `*`

**通配碼錨查詢**：
形如 `{通配}{連續碼}{參考字}`（如 `?30人`）；尾漢字左側係**數字碼**（唔係通配符）時，該字係**參考字**（佔該音節槽、同韻），**唔**因中格字面規則喺其前插 `*`。詞長 = **1（首音節通配）+ 碼位數**；連續碼約束**第 2…N 音節**（`?30人` → 三字）。四字 → `?30*人`。**中間夾 `?`**：`?3?0人`、`?30?人` 等左至右掃描。**硬切換**廢止舊「逐字元四格」`?30人`。
_Avoid_：把尾參考字當多佔字面格、喺 `?30人` 變 `?30*人` 當三字、用串長推詞長

**碼＋參考字＋通配（中格韻）**：
形如 `?{碼位}{參考字}=?`（如 `?3人=?`）→ **三音節**：首通配、中格**碼位＋同參考字韻母**（「人」係參考字，**唔**要求字面係「人」；碼 `3` 與字面「人」（碼 `0`）互斥，故**必須**寫 `=`）。**無 `=`** 嘅 `?{碼}{參考字}?`（如 `?3人?`）視作碼位同字面參考字矛盾 → **空結果 + 一行 hint**（提示改用 `?3人=?`）；唔 normalize、唔插 `*`。參考字左邊係碼時唔套用「通配符左側才 normalize `*`」規則。
_Avoid_：把 `?3人?` 當四字逐字元、把 `?3人?` 當 `?3*人?`、省略 `=` 期望中格同韻、靜默空結果唔提示

**星號錨**：
用 `*` 連接碼段與錨字，或標示**多一個音節槽**／頭格錨。**slot 連接符**（規範形 `*`）**右邊必須緊接漢字參考字或粵拼錨**（拉丁聲母／韻母／音節片段）；**唔**可以緊接 0243 碼數字——例：`2*好*人` ✓、`2*好*3` ✗ → **空結果 + hint**（例：「`*`／`+` 後須接漢字或粵拼錨，唔可以接碼。例：尾格用 `2*好3` 或 `2*好人`。」）。**唔**將 `{碼}{韻母／音節}` 二字碼粵拼錨誤當任意 `{碼}*{粵拼}`——**例外**：**三字**碼尾 `{碼}{+｜*}{韻母}`（如 `23+o` ≡ `23*我=`，**唔**等同二字 `23o`／`23我=`）。**輸入別名**：半形 `+`（U+002B）、全形 `＋`（U+FF0B）、全形 `＊`／`﹡` 與 `*` **完全等價**（**唔**收 `﹢`、`⊕` 等其它 plus 字元）；**查詢分派** normalize → 規範形 `*`。涵蓋：碼尾／中格／頭格星號錨、通配碼錨擴槽（如 `?30+人`）、首格／中格 normalize 插入（如 `?+就=`、`+門0`）、**缺字型粵拼錨**（見 **粵拼錨** § `+` 適用範圍）。**唔**把 UI **查詢分頁**開新分頁嘅 **+** 掣當查詢運算符。**只禁相鄰**多個 slot 連接符（**5a**）：全串可有**多個**非相鄰連接符（如 `*香?30+人`）；**唔支援連續** slot 連接符（如 `++`、`**`、`+＊` normalize 後相鄰多個 `*`）：與 **`?` 通配音節槽** 語意重疊；要多一個通配音節應寫 **`?`**——例 **`?30?+人`**（唔好 `?30++人`）。**連續連接符** → **空結果 + 一行 hint**（例：「唔支援連續 `*`／`+`：通配音節請用 `?`，slot 連接符最多一個。例：`?30?+人`（唔好寫 `?30++人`）。」）。Legacy 碼尾 `&`／`·` 喺 `{數字}{sep}{尾}` 仍映射 `*`。
**音節計數**：左至右掃描時，**每出現一個 `*`（或等價別名），相對於「若無該符」就多一個音節槽**（`?就=` 無 `*` → 一音節；`?*就=` → 二音節；`?30人` → 三音節；`?30*人` → 四音節）。**例外**：句首頭格 `*{字面}…` 嘅 `*` 係頭格連接符（標首音節字面或韻／聲錨），**唔**多一音節；`X…` 與 `*X…` normalize 後同一 MatchSpec（見**缺字查詢**／**韻／聲錨**）。尾格：`{碼}*{參考字}`（如 `30*人` 三字；`?30*人` 四字）。中格：`{左碼}*{參考字}{右碼}`。頭格擴展：`*{字面}{通配碼錨餘段}`（如 `*香?30人`）。`?*{參考字}=` → 二字（首通配、次同韻）。
_Avoid_：把 `?30人` 與 `30*人` 當同一詞長、把 `?就=` 與 `?*就=` 當同一查詢、頭格 `*` 當多一音節、把 `*門=0` 同 `*門0` 當等價、把查詢分頁 **+** 同星號錨混淆、把 `+` 當規範寫法（規範仍係 `*`）、用 `++`／`**` 代替 `?` 通配槽、把 `?30?人`（無 slot 連接符）同 `?30?+人` 混談、`*` 後直接接數字碼（如 `2*好*3`）

**單字韻錨查詢**：
形如 `{參考字}=`（**無** `*`、**無**前導碼）→ **詞長 1**；該字韻母同參考字（如 `就=`）。**過渡期** `?{參考字}=`（單字、無 `*`）normalize → `{參考字}=`（前導 `?` **冗餘**，語意完全等價）。與 `?*就=`（二字、首格通配＋次格同韻）完全分開。
_Avoid_：把 `就=` 當 `?*就=` 嘅簡寫、把 `?就=` 當規範形、當通配碼錨、用串長推詞長

**韻／聲錨**：
在缺字查詢基礎上，以 `=` 標定參考字音節韻／聲。**首格**：`香=?` ≡ `*香=?`、`=香?` ≡ `*=香?`（normalize）。**中格／尾格韻錨**：僅漢字**左邊係通配符**時 `?{字}=?` 與 `?*{字}=?` 等價 normalize（如 `?你=?` → `?*你=?`）；左邊係碼則走通配碼錨。**單字** `{字}=`（`?{字}=` 冗餘 normalize）；**二字** `?*就=`。**尾格同聲** `?=就` → `?*=就`（左為通配時 normalize）。
**前綴通配等號查詢**（I3）：`?{參考字}=` 且 **len(參考字)≥2** → **詞長 1+len**；首音節通配，其餘音節跟**等號查詢**整段韻／聲模板（如 `?困潦倒=` → 四字、第 2–4 音節同「困潦倒」韻；`?香港=` → 三字）。**唔**收 `?`+碼夾（`?2我=3`）、**唔**與單字 `{字}=` 混（`?就=` normalize → `就=`）。要碼約束用 `{左碼}{參考字}=`（如 `0449困潦倒=`）或**串列**。
**串列韻／聲錨查詢**（多錨）：左至右掃描；**連續數字**每位各佔一音節碼；**`?` 只可作單音節通配槽**（H1，如 `?4困=4潦=9倒=`）。每音節 **`=` 永遠喺該格碼右側**；**參考字喺 `=` 左** → 韻母錨 `{碼}{字}=`；**參考字喺 `=` 右** → 聲母錨 `{碼}={字}`。多錨例（四字碼 `0/4/4/9`）：**韻** `04困=49倒=`；**聲** `04=困49=倒`。**分派（G2）**：**≥2 參考字**走串列；**單參考字**夾碼仍走**碼夾等號查詢**。**硬切換（E2）**：觸發串列時**取代**舊 `{碼段}{字}=` 讀法（`4困=` → 一字；`04困=` → 二字第 2 格韻）。串列**唔收** `_`／`%`／`*`／`@`（`?*就=` 等走原有分派）。**純漢字多錨**（錨前無碼）→ 不合規 + 固定 hint（附例 `04困=49倒=`）。
_Avoid_：把 `*香=0` 同 `*香=?` 混談、混淆 `?就=` 與 `?*就=`、對 `?30人=` 喺「人」前插 `*`、把串列多錨當碼夾等號、混韻母錨同聲母錨於同一串、純漢字 `{字}{字}=`、用舊 `{碼段}{字}=` 三字讀法解讀 `04困=`

**三格韻錨查詢**：
三字、前後通配、**中格**讀音約束。**漢字中格、左為通配**：`?*{參考字}=?`（如 `?*港=?`）；`?{參考字}=?` 等價 normalize。**漢字中格、左為碼**：`?{碼位}{參考字}=?`（如 `?3人=?`），見**碼＋參考字＋通配（中格韻）**；`?3人?` 無 `=` 無結果。粵拼錨 `?yut?`／`?syut?` 維持。
_Avoid_：把 `?港=?` 當規範寫法、混淆 `?yut?` 與 `?syut?`、把 `?3人?` 當有效查詢

**碼字查詢**：
以連續 0243 碼開頭、夾或接參考字的查詢（如 `23就`、`20人`）；**連續數字位數 = 詞條音節數**（`20人` → 二字、整詞碼 `20`），尾隨漢字為參考字（該格字面或與參考字同韻）。查詢串**唔含**前導 `?` 時走此路徑；`?30人` 等改判**通配碼錨查詢**。碼後 `*` 接參考字屬**星號錨**（如 `30*人` 三字）。
_Avoid_：混合查詢、hybrid、假設 `?30人` 與 `20人` 只差一個 `?` 而詞長不變

**粵拼查詢**：
以粵拼輸入查詞條粵拼（如 `syut`、`nei hou`、`ming4 baak6`）。**無聲調數字**：查詢拆成音節序列，逐音節比對粵拼、忽略聲調；音節數須等於詞條字面音節數；單音節無調只返單字（如 `syut`→「雪」）。**有聲調數字**：查詢每一音節均帶調時，詞條粵拼須與查詢完全一致（僅忽略大小寫與空白差異；如 `ming4 baak6`→「明白」）。**混合有調／無調**：逐音節分級——有調音節須粵拼與調值皆合；無調音節只比粵拼、忽略調（如 `nei5 hou`→「你好」）。符合嘅**每一條讀音列**都返（同一字面可多列）。**近反義模式**不收粵拼查詢（只收漢字；輸入粵拼時應提示改模式）。0243模式與 02493模式結果相同。不做子字串模糊搜尋。
_Avoid_：粵拼片段、模糊粵拼、jyutping fragment

**等號查詢**：
整詞逐音節同韻或同聲。`詞=`（如 `香港=`）比韻母；`=詞`（如 `=香港`）比聲母。參考字至少二字；單字請用韻／聲錨（如 `=香?`）。
_Avoid_：等號韻（作查詢類型名時）、模板查詢、把 `=香` 單字當整詞等號

**碼夾等號查詢**：
形如 `{左碼}{=?}{參考字}{=?}{右碼}` 的等號查詢（如 `2=我3`、`23=你4`、`34=我`、`03=催是`）。須**同時**滿足兩類約束：(1) 整詞 0243 碼等於左碼＋右碼（無右碼時整詞 code 即左碼）；(2) 參考字在錨位只提供**一維**——`=` 在左比聲母、`=` 在右比韻母，不要求結果該格字面是參考字本身。錨位：`len(左碼) − len(參考字)`（`23=你4` → 第 2 格比聲母「你」的 `n`；`34=我` → 二字、code `34`、末格同聲「我」；`03=催是` → 二字、code `03`、整詞同聲「催是」，與詞庫詞 `03=隨時` 等價若音節拼接讀音一致）。三字同聲請用碼字查詢 `34*=我`，勿與碼夾等號混淆。
_Avoid_：用參考字整段讀音代替碼位聲調約束、對多讀音參考字做 OR 比對

## 詞庫與排序

**詞庫**：
決定哪些字面是「真詞」、以及詞級讀音（字面＋粵拼＋0243 碼）的權威來源。搜尋用的詞條庫是詞庫的副本；注入須經**收錄決策**（單字 rime、多字詞級標音、或音節拼接讀音），禁止憑空造字或繞過收錄決策猜音。
_Avoid_：把自動猜讀當詞庫、測試用造字進正式收錄

**詞庫埠**：
詞庫資料的抽象邊界：**raw lookup**（rime 單字表、詞級標音 JSON，不帶收錄判斷）與**排序信號**（essay 詞頻、curated、pron_rank）。**收錄門檻**（能否注入、讀音歸哪一類）由**收錄決策**單獨判定，不由埠的 lookup 方法隱含決定。
_Avoid_：把 runtime 詞條庫本身稱作詞庫、把收錄決策塞進 raw lookup 回傳值

**詞庫快取索引**：
詞條庫在 runtime 記憶體中的可查索引與就緒狀態；就緒後供搜尋路徑依字面形狀與聲韻維度取向量候選，未就緒時搜尋降級直查詞條庫。**離線啟動預載**的編排與磁碟暖啟屬餵入此索引的實作細節，不與索引介面混稱。
_Avoid_：word_cache（作領域正名）、把預載執行緒或磁碟快照當成索引定義的一部分

**多字收錄**：
長度 ≥ 2 的字面，僅當詞庫有該詞的**詞級標音**（粵拼＋0243 碼）時，才以詞級標音**收錄**進詞條庫；可有多個讀音列。詞級標音未命中時，不由此路徑注入（改走**音節拼接讀音**收錄路徑，若拼接成功）。
_Avoid_：逐字拼接多音字讀音後冒充詞級標音、把拼接結果稱作詞級標音

**音節拼接讀音**：
詞庫無該多字詞級標音時，逐字取 rime 單字「預設」讀音拼成整詞粵拼與 0243 碼；用於查詢、結果頁顯示與參考字錨點解析。拼接成功時可作**收錄決策**的第三條路徑寫入詞條庫，但**不**宣稱為詞級標音、**不**與詞級標音混稱。任一字缺 rime 預設則拼接失敗；單字有多個 rime 讀音時**只取預設**，不展開讀音組合。
_Avoid_：與詞級標音混稱、把拼接結果當詞庫權威、線上 API 猜音

**單字收錄**：
長度 1 的字面，詞條庫已有則不重複注入；否則從 rime 單字表取「預設」讀音注入。
_Avoid_：為排序而單獨收錄 essay 裡出現但未收錄的字

**Essay 語料**：
粵語書面／口語詞頻語料（[rime-cantonese](https://github.com/rime/rime-cantonese) · [essay-cantonese.txt](https://github.com/rime/rime-cantonese/blob/main/essay-cantonese.txt)，CC BY 4.0），格式為「詞＋詞頻」。**不**開注入門、**不**用於猜讀音；僅供排序。語料隨 repo 提供。
_Avoid_：把語料當詞庫、把語料當近義來源

**Essay 詞頻索引**：
啟動時自 Essay 語料建立的**字面→詞頻**查表，僅供搜尋排序；載入記憶體，不寫入詞條庫、不另建詞頻表。更新語料檔後重啟即生效。
_Avoid_：詞頻表（作資料庫實體）、把詞頻存進詞條列、用 vector embedding 取代詞頻排序

**Curated 常用詞**：
專案維護的填詞常用字面列表，在 essay 詞頻相同時作排序加成。
_Avoid_：把 curated 排在 essay 之前（現行 tier 為 essay 優先）

**pron_rank**：
rime 單字表的讀音常用度標記（預設＞常用＞罕見），用於單字讀音選擇與排序 tie-break；多字讀音以詞級標音為準。
_Avoid_：把 pron_rank 當收錄門檻

**詞條排序信號**：
**搜尋結果排序**與**等號參考讀音選列**共用的可比較信號定義（純漢字、Essay 詞頻、curated、pron_rank、字面等）；兩條契約 tier 順序不同，但信號來源一致，由同一模組對外提供兩條 key。
_Avoid_：把兩條契約收成一條 tier、essay_sort（實作名）

**等號參考讀音選列**：
多讀音字面在**等號參考讀音**情境下選權威單列的 tie-break：**pron_rank → Essay 詞頻 → 略過 aa 變體 → 粵拼序**；信號來自**詞條排序信號**，tier 順序與**搜尋結果排序**不同。
_Avoid_：與搜尋結果排序共用同一 tier 順序、把選列規則散落各呼叫端

**詞條 lookup 版面**：
純漢字精確查詢（詞條 lookup）回傳的多段結果編排：code header → 粵拼 header → 查詢詞本身 → 同碼同韻段（共用字字優先）→ 尾字韻錨段 → 同碼餘下；各詞條段內套用**搜尋結果排序**。快取暖化不藏在版面模組內，由呼叫端明示執行。
_Avoid_：code_aware_ranker（作領域正名）、在版面模組內隱式 sync 詞庫快取索引

**缺字查詢字面優先**：
**缺字查詢**結果在同一匹配層級內，先比**參考字**位置的字面吻合數（愈多愈前），再套用**搜尋結果排序**；不得在字面優先 tier 內另訂 essay／curated 順序。
_Avoid_：mask_priority（作領域正名）、把字面吻合與 essay 詞頻混為同一維度

**搜尋結果排序**：
扁平詞條清單在同一匹配層級內，依**詞條排序信號**套用 tier：**純漢字字面 → essay 詞頻 → curated → pron_rank → 字面**。**缺字查詢字面優先**、code header、近反義分組等結構性區塊順序不變，只在區塊內套用上述信號。目標是貼近 0243 常用度體感，不要求與 0243.hk 逐詞一致。
_Avoid_：預設字母序、每種語法各一套 tie-break、混雜粵拼字面與純漢字無區分、維護 0243 完整快照作 CI 門檻

## 產品邊界

**離線單機交付**：
本工具的主要形態：創作者在本機完整使用查韻、搜尋與近反義，詞庫與快取皆在本機，不依賴常駐遠端服務。
_Avoid_：把可選自架服務當與離線版並重的一級產品

**免安裝交付**：
創作者**無需自行安裝 Python 或執行 pip**，解壓或安裝後雙擊即可進入查韻介面；依賴與詞庫在發佈時已就緒，無需手動建庫。Windows 與 macOS 為創作者一級通道；Linux 不在此承諾內（見 **Portable 套件**）。
_Avoid_：把「仍需本機 Python」的套件稱作免安裝、把 Docker 映像當創作者免安裝交付

**Portable 套件**：
**離線單機交付**的發佈物，內含詞庫、前端與本機 API。創作者路徑：**Windows** 為解壓 zip 後雙擊啟動；**macOS** 為 **macOS 應用程式套件** 雙擊啟動——兩者皆屬 **免安裝交付**。**Linux** 仍提供啟動腳本與說明，但須本機已有 Python，不列為免安裝承諾。
_Avoid_：把 zip／tar 當產品正名、把 portable 當雲端版、要求 Linux 與 WM 同等免安裝投入

**macOS 應用程式套件**：
macOS 上創作者雙擊圖示啟動查韻介面的 `.app` 形態；與 Windows zip 並列為 **免安裝交付** 的主要通道，取代「解壓後須在終端機 chmod」作為預設創作者體驗。
_Avoid_：把 tar.gz 當 macOS 創作者預設下載、把 .app 當雲端或自架服務

**全量發佈**：
程式、依賴、詞庫與雙平台 **Portable 套件** 一併更新的正式發佈；創作者下載的 zip／`.app` 與 `lyrics.db`、`words-lexicon.json` 屬同一版本線。
_Avoid_：把每次 commit 都當全量發佈、把只換詞庫當全量發佈

**詞庫發佈**：
只更新 `lyrics.db` 與／或 `words-lexicon.json`、程式與依賴不變的發佈；須在**同一 semver** 上已完成 **全量發佈**（該 Release 已含雙平台 Portable）之後才可進行。**免安裝交付** 的 Portable 套件沿用該 semver 已發佈的 zip／`.app`，創作者只需在該版 Release 重新下載詞庫檔或自行替換本機 db。
_Avoid_：詞庫發佈仍強制重建雙平台 zip、在未完成全量發佈的 semver 上單獨發詞庫、為詞庫更新另開 semver tag 令創作者搵錯下載頁

**開發容器環境**：
供維護者與貢獻者取得一致本地開發環境的 Docker 設定；**非**創作者交付物，不取代 **離線單機交付** 或 **免安裝交付**。
_Avoid_：把 Docker 當創作者主要安裝方式、與 **自架共用服務** 混為一體的一級產品

**Ingest 執行狀態**：
本機執行 ingest 時產生的 lock 與 checkpoint，僅供該機器防重入或續跑長任務；不納入版控、不隨 Portable 交付。
_Avoid_：把 checkpoint 當詞庫資料、把 lock 檔 commit 進 repo

**源碼根目錄**（維護者）：
Clone 後第一層目錄；保留產品入口、`CONTEXT`、變更紀錄（`WORKLOG`）、agent 入口 stub（`AGENTS`）與 agent 鎖定（`skills-lock`）；對外授權與第三方聲明留根目錄；貢獻流程等維護說明收在 `docs/`；開發啟動用 `start.sh` 與 Portable 套件內啟動腳本並存、職責分開。根目錄允許的本機檔僅 `lyrics.db` 與 `.env*`（範例為 `.env.example`）；日誌、debug 與 **Ingest 執行狀態** 不得停留或進版控。
_Avoid_：把根目錄當資料 dump、與 Portable 解壓目錄混淆、把開發啟動腳本與 Portable 啟動腳本混為一體

**建置暫存**（維護者）：
本機 `dist/` 目錄，存放 build 腳本產出的 zip、解壓樹、`words-lexicon.json` export 等；**不進版控**、**非**創作者下載來源。正式 **全量發佈** 四件套以 GitHub Release 為準；Release 確認後可整目錄刪除，需要時再跑 build／export 重建。
_Avoid_：把本機 dist 當發佈歸檔、在 dist 內長期保留多版 zip、把 dist 解壓樹當日常開發工作目錄

**本機啟動**：
維護者 `./start.sh` 與創作者 **Portable 套件**內 `START.bat`／`START.sh`／`.app` 共用同一啟動契約：（1）**終端／視窗須在執行后立刻**顯示「正在啟動」類回饋（Windows 英文、其他平台可中英並列），避免雙擊後數秒無任何可見反應；（2）背景程序就緒且**查韻介面 HTML 可載入**後儘快開啟瀏覽器，**不**等待**就緒閘**解鎖；（3）頁面由既有就緒閘接手進度與搜尋解鎖。不得出現瀏覽器原生「無法連線」畫面——須確保 HTTP 已能回應查韻介面後才開啟 URL。維護者應在開發版啟動路徑驗證與 Portable 相同的體感延遲（含 HTML 就緒時間），不可只在 zip 內手測。
_Avoid_：等 gate_ready 才開瀏覽器、先開瀏覽器再等後端（未就緒時 fetch 失敗）、開發版與 Portable 啟動行為長期分叉、雙擊後長時間無終端回饋

**Windows 啟動訊息**：
Portable 在 Windows 上啟動時，終端機提示以**英文**呈現，避免部分系統語系下中文無法正確顯示；`README` 與 macOS／Linux 啟動腳本仍可中英並列。
_Avoid_：在 Windows 啟動腳本內寫中文 echo、要求使用者先讀 bat 才懂怎麼用

**自架共用服務**：
多人透過遠端 API 共用同一詞庫的部署形態；**非**目前主要交付目標；相關程式屬凍結、不主動投資。
_Avoid_：Postgres 版、hosted 版（作產品名稱時）

**創作者**：
使用本工具搜韻、換字、查近反義以完成歌詞、文案或其他粵語創作的人。
_Avoid_：填詞者（範圍過窄時）、用户、user

**首次進入**：
同一 browser session 內第一次開啟查韻介面；同分頁重整不視為首次進入。
_Avoid_：跨 session 的「首次訪客」、first visit（實作名）

**就緒閘**：
**詞庫快取索引**預載完成前阻擋搜尋的過渡狀態；須向創作者傳達載入進度，就緒（或**降級逾時**）後才解鎖搜尋列。凡經**查詢分派**的搜尋請求，在閘未解鎖前應拒絕，與 UI 一致，不得僅靠前端隱藏搜尋列或個別路由各自判斷。
_Avoid_：splash screen、把就緒閘當純裝飾開場、就緒與搜尋可用性脫節、預載中仍允許搜尋 API 靜默降級直查詞條庫

**離線啟動預載**：
離線查韻 runtime 啟動時載入的靜態詞林、排序語料、**詞庫快取索引**與複合詞預算；就緒閘以**詞庫快取索引**就緒為準，其餘資源可 eager 或首次使用時載入。
_Avoid_：startup hook（作領域正名）、把 lifespan 當產品概念

**非首次進入**：
同 session 內再次開啟查韻介面（含分頁重整）；若詞庫仍需預載則顯示精簡就緒閘，不重演完整首次進入流程。
_Avoid_：每次重整都播完整入場流程

**查詢分頁**：
創作者在同一**查韻介面**內並行維護多個查詢上下文，用以取代 0243.hk 式「網址列連開瀏覽器分頁」的工作流。各分頁獨立保存查詢字串、結果清單、分頁偏移與**分頁 view**（搜尋、教學或關係補錄）；**模式切換**與就緒閘為全介面共用。**查詢分頁列**常駐顯示（即使只有一個分頁），**+**／**×** 為開關新查詢分頁的主要操作；並提供避開瀏覽器保留鍵的快捷鍵（開新查詢分頁、關閉作用中分頁）。創作者可拖曳任一分頁標籤以**重排分頁列順序**；按住時該分頁即成**作用中查詢分頁**；**新查詢分頁**按鈕（**+**）固定於列尾、不參與重排。分頁標籤顯示截斷後的查詢字串；空白搜尋分頁顯示「新查詢」，教學與關係補錄分頁顯示對應固定標籤。頂欄**搜尋教學**與**補關係**採智慧合併：session 內該 view **至多一個分頁**——已有則切換至該分頁，尚無則開新分頁。**新查詢分頁**預設為空白搜尋並聚焦搜尋列，且不可關閉至零個分頁。分頁數量不設硬性上限；若效能不佳由創作者自行關閉分頁。點頂欄品牌返回搜尋時，僅重置**作用中查詢分頁**的搜尋內容，不影響其他分頁。網址列僅反映**作用中查詢分頁**的參數：`q`／`mode` 用於搜尋分頁；`view=guide` 為教學、`view=relation` 為關係補錄（此二 view 不帶 `q`）。同一 **browser session** 內重整查韻介面時，應還原全部分頁及其**列順序**，關閉瀏覽器後則不保留非作用中分頁。
_Avoid_：把瀏覽器原生分頁當領域正名、要求創作者複製貼上完整本機 URL、每分頁各一套模式切換、僅有多分頁時才出現分頁列、把全部查詢分頁塞進網址列、新分頁預設複製作用中查詢、用 Ctrl+T／Ctrl+W 作主要快捷鍵（瀏覽器會攔截）、品牌返回清空全部分頁、以硬性分頁上限阻擋開新查詢、拖曳 **+** 或把分頁拖到列尾之外

**降級逾時**：
就緒閘在**詞庫快取索引**長時間未能就緒時，仍解鎖搜尋的逾時策略；由 server 自預載開始計時，逾時後就緒閘解鎖並標示降級，搜尋 API 與 UI 同步放行（效能可能低於索引就緒後）。與預載 **failed** 立即解鎖並列，皆屬同一就緒閘契約。
_Avoid_：timeout（作領域正名）、僅 UI 解鎖而 API 仍拒絕、前端與 server 各算各的逾時

**模式切換**：
頂欄可一鍵切換 0243模式、02493模式、近反義模式；前兩者屬 **0243搜尋模式**，旁註「鬆」「緊」；近反義簡寫「近反義」、旁註「查」。搜尋列下方讀出格式為正名加旁註（如「目前模式：0243模式（鬆）」）。搜尋教學維持獨立入口。
_Avoid_：把三種主模式藏進需多步才能到達的選單
