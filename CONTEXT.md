# Canto-0243

離線粵語填詞查韻工具：依 0243／02493 數字碼、粵拼與韻母規則，幫創作者找出可替換的詞條與近義／反義關係。

## 如何使用本文件

本文件是**技術領域詞彙表**，不是實作規格。討論功能、測試或領域行為時，優先使用下列正名；`_Avoid_` 列出易混淆或禁用的說法。

## 核心資料

**詞條**：
字典中一條可查記錄，由字面、0243 碼與粵拼組成；同一字面可有多條詞條（多讀音）。**詞條庫**內 canonical 列以 **(字面, 粵拼)** 唯一；邊個 upstream 貢獻由 **詞條來源紀錄** 追蹤。
_Avoid_：Word（實作名）、候選（僅在描述搜尋結果清單時使用）、用 0243 碼單獨當 canonical 唯一鍵（碼由粵拼衍生）

**詞條來源紀錄**：
**詞條從源重建** 時記錄 canonical **詞條** 與 **詞條 SSOT 來源** 之間嘅 provenance；一條 **詞條** 可對應多個來源行（多源同意同一讀音）。唔 duplicate **詞條** 上已有嘅字面／粵拼／0243 碼。
_Avoid_：provenance（作領域正名）、把來源紀錄當第二套讀音倉庫、在來源紀錄重複存 jyutping

**字面**：
詞條上顯示的漢字序列（一至多字），例如「香港」的字面長度為 2。
_Avoid_：詞面、字串

**程式識別子（canto 字）**：
討論或撰寫程式時，指 CJK 字元嘅識別子分工：**`chars`** 指字元序列／字串操作／regex 範圍（如 `mask_chars`、`q_chars`）；**`canto`** 指單一粵語書面字、音節槽內一格（如 `canto_char`、`has_canto`）。領域討論、UI 文案同 hint 仍用**漢字**／**字面**；`canto`／`chars` 只限程式命名。歷史識別子 **`literal_char`**（MatchSpec slot kind）語意等同固定 canto 槽，保留唔改；新 code 唔再新增 `*_char` 識別子。
_Avoid_：`hanzi`、`Hanzi`、`_HANZI_*`、把 `chars` 同 `canto` 混用於同一概念、喺用戶文案改用 `canto`/`chars`、新 code 用 `*_char` 命名（`literal_char` 除外）

**0243 碼**：
附在詞條上的數字序列；每一格對應一個音節的聲調，經固定映射轉成 0/2/3/4/9。長度與字面音節數一致（「香港」→ 兩位碼）。詞庫存儲一律用此規範形。
_Avoid_：code（實作名）、音碼、把 0243 碼當韻母鍵盤映射、九鍵（易與電話鍵盤混淆）

**02493 碼**：
0243 理論延伸嘅聲調數字序列；查詢可含 **0243 碼**以外嘅數字（如 `1`、`5`、`6`、`7`、`8`），以分清二聲等細分。比對前須**正規化**成 **0243 碼**再對詞庫；**0243搜尋模式**內凡作聲調數字出現嘅位（純數字、混合碼字、串列錨、近反義前綴碼等）皆適用，單一規則入口。**0243模式**另加鬆檔等價變體，**02493模式**唔加。逐位映射：`1→3`、`5→4`、`6→2`、`7→3`、`8→4`；`0`／`2`／`3`／`4`／`9` 不變。映射表已完整（`0`–`9` 全覆蓋）。**0243搜尋模式**內零結果：**無其他 hint** 時只顯示粗體「搵唔到「{查詢}」。」（唔加第二行）；**已有其他 hint** 時第一行「搵唔到」、第二行保留原 hint。
_Avoid_：9鍵、九鍵、9-key（實為 9 **碼**／數碼，唔係電話九宮格）、把 02493 碼當詞庫存儲格式

**粵拼**：
詞條的粵語羅馬字標注，一音節一段（如「香港」→ `hoeng1 gong2`），與 0243 碼逐音節對應。
_Avoid_：jyutping、拼音（易與普通話混淆）、拉丁、拉丁字母、字母（作產品或教學文案時）

**讀音**：
詞條上完整音節的粵拼（如「你」→ `nei5`）。查詢比對時，參考字多半只提供**聲母或韻母其中一維**作錨；0243 碼數字則單獨約束該格聲調，與參考字是 `nei5` 還是 `nei1` 無關。
_Avoid_：把參考字的整段讀音等同於該格的查詢條件

**聲母**：
粵拼音節的輔音部分（如 `gong2` 的 `g`）；同聲比對以聲母為單位。
_Avoid_：initial、字頭

**韻母**：
粵拼音節的韻部（含調，如 `gong2` 的 `ong2`）；同韻比對以韻母為單位，與 0243 碼的聲調維度不同。
**y- 韻核**：辅音（叢集）後接 `y` 再接元音時（如 `jyu`、`zyu`、`syu`），`y` 與後續元音屬**韻母**（如 `yu`、`yut`），唔併入聲母叢集（如唔把 `zyu` 拆成聲母 `zy`、韻母 `u`）。
**u／yu 韻部**：`u` 與 `yu` 為不同韻母鍵（如 `fu` 與 `zyu`／`住`）；同韻比對**唔**跨系匹配。
**獨立鼻音韻母**：粵拼**整個音節**僅為 `m` 或 `ng`（加調，如唔 `m4`、吾 `m4`、五 `ng5`、午 `ng5`）；該字面**無聲母**，查詢比對上**只**屬韻母維度；`m` 與 `ng` **完全等價**（mrpinyin M／NG 欄）。唔同「門」`mun` 嘅輔音聲母 `m`，亦唔同「我」`ngo` 嘅聲母叢集 `ng`。
_Avoid_：final、字尾、把唔／五／午等獨立鼻音音節當有聲母、把 codec 存入 `initials` 嘅 `m`／`ng` 當聲母錨匹配、把 `zyu` 韻部當 `u`、用韻母粵拼錨 `u` 匹配 `yu` 音節

## 比對關係

**同韻**：
兩個字面在對應音節位置上韻母一致；常以 `=` 或參考字標示（如 `就=`、`香港=`）。
_Avoid_：押韻（指比對規則時）、rhyme

**同聲**：
兩個字面在對應音節位置上聲母一致；常以 `=` 與參考字標示（如 `?=就`、`=香?`）。
_Avoid_：用「同音」泛指同韻或同聲、initial（實作名）

**同音**：
在本工具中，指對應音節上的 0243 碼與查詢數字一致（0243模式下含等價變體）；不指整個音節聽感完全相同。
_Avoid_：用「同音」泛指同韻或同聲；同碼（若 UI 已用「同音」則不必並列）

## 搜尋模式

**0243搜尋模式**：
以 0243／02493 數字碼為核心的搜尋，含 **0243模式** 與 **02493模式** 兩檔；與 **近反義模式** 並列為頂欄三大 pill 中之兩檔。
_Avoid_：鬆／緊並稱作模式正名、把 0243搜尋模式當成近反義的同義詞

**0243模式**：
0243搜尋模式預設檔：0243 碼比對時允許等價變體。介面選單旁註顯示「鬆」。
_Avoid_：鬆模式（作領域正名）、m1（實作名，僅程式與 URL）

**02493模式**：
0243搜尋模式進階檔：接受 **02493 碼**查詢並**分清二聲**（如 `9` 與 `3` 唔再合併）；比對前仍將 **02493 碼**正規化為 **0243 碼**，但**唔**套用 **0243模式**嘅鬆檔等價變體。介面選單旁註顯示「緊」。
_Avoid_：緊模式（作領域正名）、m2（實作名，僅程式與 URL）、9鍵／九鍵（應作 9 碼）

**近反義模式**：
以詞義關係為主的搜尋，回傳近義、反義與語意相關詞條。介面 pill 可簡寫「近反義」；旁註顯示「查」。教學與文件用全名。輸入**近反義關係查詢**語法時觸發**搜尋模式轉接**（見下），唔再空結果。
_Avoid_：syn、查模式（作為模式正名）、關係模式

**近反義前次0243搜尋模式**：
用戶切換至**近反義模式**之前所選的 **0243搜尋模式** 檔（**0243模式**或 **02493模式**）；**搜尋模式轉接**時還原此檔執行**近反義關係查詢**。若無紀錄（如首次進站即在近反義），預設 **0243模式**。**API 保險轉接**時可由客戶端以 query param `fallback_0243_mode` 傳遞（優先）；同名 HTTP header 可選；皆缺則預設 **0243模式**。
_Avoid_：lastMode、m1/m2（實作名）

**搜尋模式轉接**：
在**近反義模式**下，若查詢串屬**近反義關係查詢**語法，自動切換至**近反義前次0243搜尋模式**（無則 **0243模式**），更新頂欄模式與 URL，並以該檔執行查詢；純漢字池查詢（如「開心」）仍留喺近反義模式。轉接時重設分頁並顯示**轉接提示**。執行分兩層：**介面轉接**（送出前先切模式）與 **API 保險轉接**（後端改檔並回傳 `hint`／`X-Effective-Mode`）。
_Avoid_：模式偷換（UI 仍顯示近反義）、把缺字／碼字查詢一併轉接、僅後端轉接而介面唔更新、轉接後沿用舊 offset

**轉接提示**：
**搜尋模式轉接**發生時，於結果區頂部顯示嘅一行說明（如「此語法已切換至 0243模式（鬆）查詢」）；與查無結果、粵拼在近反義模式被拒等既有 `hint` 同列，唔阻塞結果展示。**介面轉接**由前端本地生成；**API 保險轉接**由後端 `hint` 回傳；兩層文案一致。
_Avoid_：toast、彈窗確認

**語意相關**：
與查詢詞在語意上有關聯、但非近義亦非反義的詞條（近反義模式第三類結果）。
_Avoid_：相關詞（不區分類型時）、semantic_related

**近反義池**：
對一個字面，合併 `word_relations` 與**靜態詞林**後、經排序與去重的近義、反義與語意相關候選集合；候選必須是**有效字面**（含漢字、無詞林編碼或英數噪音）；**近反義模式**與**近義橋反義** ingest 共用同一建池規則。
_Avoid_：relation pool（作領域正名）、syn_ant_service（作池名稱）

**近反義池投影**：
Runtime 讀取**近反義池**的統一入口；近反義模式、`~` / `!` 查詢與關係補錄擴展互斥皆經此投影取得池快照或字面集合，建池規則本身不變。
_Avoid_：pool facade、runtime 直呼 build_pool

**靜態詞林埠**：
bundled 近義／反義語料（詞林 cilin、國語辭典近義檔、反義語料檔）的 **raw lookup** 邊界。Runtime `~`／`!` 以合併近義、反義 getter 消費；ingest 可用分源 getter（cilin、guotong、antisem）與種子字面全集枚舉，寫入 `word_relations` 時保留來源標籤。資料載入與模組內 dict 為實作細節，對外只經埠存取。
_Avoid_：ThesaurusPort（作領域正名）、static thesaurus（作中文正名）、直接讀模組級 dict

**收錄決策**：
對一個字面，在 raw lookup 之上判定能否注入詞條庫、讀音來自何處（單字 rime、多字詞級標音、或音節拼接讀音），輸出三分類結果。**參考字讀音解析**須經此決策鏈；`ensure_word_in_db` 僅依決策結果執行 DB 注入，不重複收錄規則。
_Avoid_：admission（作領域正名）、在 ensure 層重複拼接讀音邏輯、把收錄判斷藏進詞庫埠 lookup、快取優先於收錄決策當讀音權威

**參考字讀音解析**：
依**收錄決策**解析**參考字**讀音，對外兩條入口：（1）**錨點音素選項**——單字**參考字**在**韻／聲錨**所在音節的聲母或韻母選項集合（如「香」在 `香=?` 取韻母 `oeng`，在 `=香?` 取聲母 `h`）；多讀音時 **union 全部可用選項**（如「行」同時含 `hang` 系韻部與 `hong` 系韻部）。（2）**等號參考讀音**——**等號查詢**與**碼夾等號查詢**所用整詞字面的粵拼音節序列（聲母列或韻母列）；多讀音時取 **pron_rank** 權威單列，**不** OR 比對（與**碼夾等號查詢**約束一致）。`pron_rank` 平手時依 **Essay 詞頻**（字面級）降序，再平手略過 `aa` 變體寫法（如 `haang` 視同 `hang` 族）後依粵拼字串升序選單列。呼叫端明示是否允許注入詞條庫——**查詢分派**與**缺字型查詢執行**的比對路徑可注入，**詞庫快取索引**取向量候選與預載路徑不注入。注入成功後才更新**詞庫快取索引**；解析本身不隱式 sync 既有詞條。不以快取為讀音權威。
_Avoid_：phoneme_lookup（作領域正名）、在索引候選解析時隱式 ensure 或 sync、把參考字整段讀音當比對條件、在 match_equals 重複 ensure 編排、等號路徑對多讀音做 OR

**關係寫入**：
將近義、反義列寫入詞條關係表的領域操作；含 canonical 字面對排序、收錄字面索引、去重插入。每 **字面** 以 **關係錨點詞條**（最小 id 代表列）作關係端點，**唔**對多讀音列做笛卡爾展開。Runtime 與 ingest 共用同一寫入規則；ingest 僅作批次 adapter。**現行過渡（B）**：靜態語料 normalize 後**直寫** `word_relations`，**唔**經 **關係 staging**；長遠再評估 **C**（runtime 以靜態詞林＋烘焙快照為主、縮減或廢止 `word_relations`）。
_Avoid_：merge pipeline（作領域正名）、把 ingest 套件當 runtime 依賴、staging→relations JOIN 展開同一字面全部讀音列、長期保留雙管線（staging 寫入再 JOIN）

**關係 staging**：
**詞條庫建置命令** 中間站表（`syn_ant_edges`）：存放 normalize 後嘅**字面級**近義／反義邊，供舊管線 `build-relations` JOIN `words` 寫入 `word_relations`。**Runtime 唔讀**。**已廢止**：build 熱路徑與 maintainer CLI **唔**再寫入或依賴 staging；改 **關係直寫**。表與 `normalize`→staging、`build-relations` 子命令從產品路徑移除（非 debug 保留）。
_Avoid_：把 staging 當近反義池、把 staging 當詞條 SSOT、創作者查詢依賴 staging、build-db 仍經 syn_ant_edges

**關係直寫**：
normalize 產出嘅字面級邊，於 build 時直接映射 **關係錨點詞條** id 並 bulk 寫入 `word_relations`；跳過 **關係 staging** 與 `build-relations` JOIN。與 **靜態詞林埠** 讀同一批語料，但寫入 DB 供 `in_db` 排序、衍生反義、**關係補錄** 等仍依 `word_relations` 嘅能力。**C**（runtime 以靜態詞林＋烘焙快照為主、縮減 `word_relations`）**暫緩**：B 落地並量測 build 時間／回歸後再議。**cilin** 近義須經 **leaf 解析**（同原 `ingest-cilin` 同源）以保留 `group_codes`；`build-db` **唔**再單獨跑 `ingest-cilin` 步。
_Avoid_：直寫仍經 staging、直寫展開多讀音 id 對、直寫 cilin 無 group_codes、build-db 同時直寫 cilin 又跑 ingest-cilin、未量測就廢 word_relations

**關係錨點詞條**：
**關係寫入** 時每 **字面** 揀一條代表 **詞條**（現行：該字面最小 `word.id`）作 head／tail 端點；與 **等號參考讀音選列**（多讀音揀權威列）係不同契約——關係圖以字面為鄰居，唔複製異讀邊。
_Avoid_：把關係錨點當查詢用權威讀音、對多讀音字面寫 N×M 條關係

**關係圖**：
對已收錄字面，查詢**直接近義鄰居**（`word_relations` 雙向近義 **＋** 靜態詞林近義）與**直接反義鏡射對**（`!` expand 同源）。近反義池建構、創作者手動關係擴展、近義橋反義 ingest 共用同一鄰居規則。
_Avoid_：CharRelationGraph（作領域正名）、在 ingest 繞過靜態詞林埠直讀模組 dict

## 查詢語法

**查詢分派**：
將創作者輸入分類為等號、碼字、缺字等語法類型，正規化為可比對規格並路由至對應搜尋路徑；**分派優先序**本身是領域規則的一部分。執行層（**缺字型查詢執行**）不應重複語法解析或 ParsedQuery→規格轉換。
_Avoid_：parser、handler chain（實作細節）、在 position_match 各處散落 build_* 正規化

**查詢種類**：
`parse_query` 產出後，每條查詢歸一類（等號、串列韻／聲錨、前綴通配等號、詞條 lookup…）。**比對規格建構**與**搜尋路由**須以同一**查詢種類**驅動，唔再各自維護第二套型別分支；**parse_query** 優先序鏈可維持現狀。
_Avoid_：parse 後仍用 dataclass isinstance 梯做 spec／dispatch、把查詢種類當實作 enum 名寫進領域文案

**查詢語意解釋**：
向**創作者**即時說明當前搜尋字串嘅查詢意圖；權威來自**查詢分派**解析，唔另建前端 parser。**等號查詢**（整詞同韻／同聲）用一句概括（如整詞同「香港」同韻），句尾括**押韻標註**（單押／雙押／三押／四押，依參考詞字數）。**前綴通配等號查詢**先述首個字**任意字**，再述第 2 字起同參考詞讀音模板**同韻**並附押韻標註。其餘缺字家族等以**由左至右**逐字掃描組句：先標**幾個字**，再逐個字列出約束（如第 N 個字同 {碼} **同音**、第 N 個字為「{字面}」、第 N 個字**任意字**、第 N 個字同「{參考字}」**同韻**／**同聲**、第 N 個字同{片段} **同韻母**／**同聲母**）。**詞條 lookup**、**近反義關係查詢**等無逐字比對規格者用最短動作句（如查詢詞條「香港」、查「開心」嘅**近義詞**／**反義詞**）。**位置錨定**易混時另加簡短警告，唔取代主解釋。語法無效時沿用 parser 已有 hint。
_Avoid_：格、槽、音節、音節槽（作創作者解釋文案）、前端複製 parse 規則、把解釋當第二套語法規格、「同音」泛指同韻或同聲、押韻標註用於逐字掃描句、解釋文案寫近義關係／反義關係（改用近義詞／反義詞）、別名或等價查詢（如 `23我` 與 `23我=`）解釋與實際分派唔一致

**押韻標註**：
**等號查詢**語意解釋句尾括號；**前綴通配等號查詢**句尾亦用同一套標數，標示參考詞字數與填詞習慣：一字（單押）、二字（雙押）、三字（三押）、四字（四押）；五字及以上依字數延伸（五押、六押…）。含左碼整詞等號（如 `0449窮困潦倒=`）時，押數仍跟參考詞字數，碼約束另句補充。
_Avoid_：當通用韻腳標籤用於缺字逐字掃描句或**韻／聲錨**單字查詢、與搜尋結果排序混淆、cap 四押

**缺字型查詢執行**：
接收**查詢分派**正規化後的比對規格，將缺字查詢、韻／聲錨、碼字查詢、字面參考、碼夾等號查詢，以及標有 `compound_kind` 的近義／反義複合查詢，統一執行為搜尋結果；不應再依 ParsedQuery 型別分支或重複語法正規化。語法上 ~~／!! 仍不屬缺字型查詢家族。
_Avoid_：位置查詢、position match、候選來源（作領域正名）、在執行層保留 isinstance 分派梯

**近反義關係查詢**：
在 **0243搜尋模式**下以 `~`（近義）或 `!`（反義）查詞義關係；可帶前綴 0243 碼（如 `33!開心`）。`!!`／`~~` 為複合詞；`!{連接}!`／`~{連接}~` 為**連接詞複合查詢**。於**近反義模式**輸入時觸發**搜尋模式轉接**。全形 `～`／`！`／`＄`／`／` 與半形 `~`／`!`／`$`／`/` 等價；`？` 與 `?` 等價——入口正規化為半形後再解析。
_Avoid_：稱此為「開啟近反義模式」（那是近反義模式的另一條路徑）

**反義複合詞**：
填詞二字對舉（如「生死」「是非」）；`!!` 及 `33!!`、`!!你` 等變體，只返詞庫 2 字詞。候選：curated ∩ 詞庫（AB／BA 對稱）→ 反義素掃描 → 單字反義合成（top‑K=12）；啟動預算前两源，合成查詢時追加。
_Avoid_：一般雙字詞、把 `!!` 當僅 curated 列表

**反義複合快照**：
**離線啟動預載**就緒後供 `!!` 讀取的候選字面（curated ∩ 詞庫 + 反義素掃描）；單字合成不在快照內。
_Avoid_：compound_ant_cache（作領域正名）、把查詢合成寫回快照

**填詞連接詞**：
**連接詞複合查詢**中格固定 11 字：**與、和、或、共、同、及、跟、而、且、並、向**。
_Avoid_：把任意虛詞都當填詞連接詞

**連接詞複合查詢**：
三字；中格為**填詞連接詞**；首尾互為反義（`!{連接}!`）或近義（`~{連接}~`）；可帶尾韻錨。只返詞庫已有三字詞；`A{連接}B`／`B{連接}A` 對稱。
_Avoid_：把 `!與!` 當整詞 lookup、無首尾關係約束

**近義複合詞**：
填詞二字近義複合（如「恐懼」）；`~~` 及變體，候選規則同**反義複合詞**對稱（curated → 同義素掃描 → 單字近義合成）。
_Avoid_：把一般雙字並列詞都當近義複合

**近義複合快照**：
供 `~~` 讀取的候選字面（curated + 同義素掃描）；單字合成查詢時追加。
_Avoid_：compound_syn_cache（作領域正名）

**源 3 tier 快取**：
首次 `~~` 查詢 lazy 預算單字近義合成 tier 表；效能優化，不擴快照，不重啟即失效。
_Avoid_：把源 3 tier 快取當快照

**近義橋反義**（ingest 補全）：
有近義無反義的字面：在近義候選中以語意向量找**橋接近義**（已有反義者）；head 與橋接近義須達**橋接語意門檻**，未達則不貢獻反義。**多橋合併**：合格橋候選皆可貢獻直接反義，去重後按**橋分**排序，取前 N（**橋接借入上限**）。寫入關係表（`source=ant_syn_bridge`）。**詞條庫建置命令**預設由 **近義橋反義快照** 注入，唔跑 embedding；embedding 全量重算僅用於**烘焙**快照。僅 ingest，不 runtime 載 embedding。**多補取向**：門檻偏寬，寧可多橋接、以筆數上限控噪音。
_Avoid_：runtime 即時猜反義、每次 build 預設跑 embedding 橋接

**近義橋反義快照**：
由 embedding 橋接**烘焙**產出、git 追蹤嘅字面級反義對列表（如 `data/syn_ant/ant_syn_bridge_pairs.tsv`）；**詞條庫建置命令**預設 ingest 此快照（`source=ant_syn_bridge`），取代日常 rebuild 跑模型。近義／反義池或橋接品質閘門變更後，maintainer **重烘焙**並 commit 新快照。
_Avoid_：把快照當未修改 upstream raw、每次 build 重算 embedding、快照唔入版控導致 CI 缺橋接

**近義橋重跑**：
近義橋反義規則或近反義池變更後之 **近義橋反義快照** 重烘焙（embedding 全量）並 commit；納入 maintainer **詞條庫建置命令** 流程。**唔**係每次 **詞條從源重建** 必做。維護步驟見 `docs/ingest-bridge-ant.md`（唔寫入創作者 README）。
_Avoid_：貢獻者 PR 內代跑 embedding 烘焙、把日常 build-db 同重烘焙混為一步、手動刪 lock 檔當恢復

**橋接語意門檻**：
近義橋反義 ingest 時，head 與橋候選近義之語意相似須達的最低要求；未達之橋候選不貢獻反義。
_Avoid_：把筆數上限當作語意門檻

**橋接借入上限**：
每個 head 經多橋合併後寫入之直接反義筆數上限（預設 30）；依橋分由高到低取前 N。
_Avoid_：把單一橋候選之反義池大小當作借入上限

**橋分**：
head 與橋候選近義之語意相似；多橋合併時作借入反義去重保留與排序依據（較高橋分優先）。
_Avoid_：把橋分當作 head 與每條反義之直接相似

**多橋合併**：
近義橋反義自多個合格橋候選借入直接反義，字面去重後按橋分排序截斷至橋接借入上限。
_Avoid_：單橋贏者通吃、以反義池 outdegree 剔除橋候選

**橋接目標排除**：
已具反義之字面不參與近義橋反義；同輪 chunked ingest 每批寫入後須反映新狀態，避免重複橋接。
_Avoid_：僅以開跑快照判定目標、手動刪 lock 檔恢復

**創作者手動關係**：
**關係補錄**寫入 direct `word_relations`（`source=manual`）並一跳擴展（`manual_syn_cluster`／`manual_ant_mirror`）；擴展互斥：唔同時寫入種子字面既有相反關係類型。**創作者**填詞時**即時寫本機 **詞條庫****（唔經 rebuild）；**維護者**將要進 Release 嘅補錄寫入 **關係補錄清單**，由 **詞條庫建置命令** 套用。
_Avoid_：手動關係（作領域正名）、稱作近義橋反義、創作者補錄只寫 TSV 要等 rebuild、把本機 DB 補錄當 Release 權威

**撤回創作者手動關係**：
**創作者**可即時撤回本機 DB 列。**維護者**從 **關係補錄清單** 刪除或標記撤回；下次 **詞條從源重建** 生效。唔改 ingest 快照或 **近義橋反義快照**。
_Avoid_：撤回 ingest 關係、維護者只改 DB 唔改清單

**關係補錄清單**：
git 追蹤嘅 **關係補錄** declarative 檔（如 `data/relations/manual_relations.tsv`）；**詞條庫建置命令** 於近反義 ingest 完成後套用，寫入 `manual` 系關係並執行一跳擴展規則。只影響 Release **詞條庫**；**唔**取代創作者本機即時補錄。
_Avoid_：把清單當 upstream raw、用 `pending`／`applied` 狀態機、與 **詞庫勘誤** 混檔、要求創作者寫清單先至查到

**關係補錄介面**：
創作者手動關係專用 view；**查詢分頁**承載於**查韻介面**。創作者操作**即時寫本機 **詞條庫****；維護者將 curated 補錄同步寫入 **關係補錄清單** 以納入下次 Release。
_Avoid_：獨立 relation-entry 頁、雙軌 HTML、創作者操作只寫 TSV

**關係補錄命令**：
套用或檢查 **關係補錄清單**；補錄或撤回；共用字面驗證與一跳擴展計畫。
_Avoid_：manual command（作泛稱）、獨立 apply 跳過 rebuild

**詞庫勘誤**：
維護者維護嘅 **詞條** 標音修正 overlay（declarative）；每次 **詞條從源重建** 必套用，優先序凌駕所有 **詞條 SSOT 來源**。以 **(字面, 舊粵拼)** 鎖定目標，**可選** **舊 0243 碼** 作第三把鎖（`set_code` **必填**）；在 SSOT 合併**之後**、寫入 **詞條庫** **之前**介入。TSV 欄：`char`、`old_jyutping`、`old_code`（可選）、`action`、`value`、`note`；**唔**含 `status`／`applied_at`。支援 `set_jyutping`（改讀音、連帶重算 0243 碼）、`set_code`（粵拼已對只改碼）、`delete`（刪該 canonical 讀音）。**唔**改 raw 檔、**唔**以狀態機決定是否套用、**唔**負責 upstream 無字面嘅**新增收錄**（見 **詞級標音 curated**）。與**關係補錄**（近義／反義）無關。
_Avoid_：把詞庫勘誤當關係補錄、只靠字面改多讀音列、用 `applied` 當權威、用勘誤 TSV bulk 補缺詞、逐源 overlay（除非另定源級審計）、`set_code` 唔鎖舊碼

**詞庫勘誤介面**：
維護者專用 **查詢分頁**；**查韻介面**搜尋框輸入 `debug` 開啟，**唔**出現喺一般導覽。填詞時把標音問題寫入 **詞庫勘誤** TSV（字面＋舊粵拼＋可選舊 0243 碼），**唔**即時改 **詞條庫**；生效須經 **詞條從源重建**。與 **關係補錄** 無關。
_Avoid_：把詞庫勘誤介面當關係補錄、創作者-facing 入口、即時改詞條庫、debug URL 門檻

**詞庫勘誤命令**：
觸發或檢查 **詞條從源重建** 流程中 overlay 段；`set_jyutping` 或 `set_code`；`delete` 刪錯列。**唔**再批次 `apply` 改現行 **詞條庫** 而跳過 rebuild。
_Avoid_：勘誤 action（作泛稱）、把重算碼叫改粵拼、獨立 apply 當權威路徑

**詞級標音 curated**：
**詞條 SSOT 來源** 之一：maintainer 維護嘅 **詞級標音** 小表，優先序在 words.hk／開放詞典之後，僅補前序仍無之字面。可 **新增收錄**；**詞庫勘誤** overlay **唔**負責新增缺詞。
_Avoid_：把排序用 curated 同本檔混為一檔、用 CC raw 自動填表、數千條無審核 bulk 匯入、新增缺詞寫入勘誤 TSV、Beta 前一次 diff 自動灌入 legacy 讀音

### 查詢語法基礎

**參考字**：
查詢語法中標定某音節位置的漢字，用於同韻、同聲或字面約束（如 `23就` 的「就」、`2=我3` 的「我」）。
_Avoid_：錨字、錨點字

**等號（韻／聲）**：
`=` 緊貼**參考字**定讀音維度：**參考字後** `字=` → 比**韻母**；**參考字前** `=字` → 比**聲母**。口訣與**加號錨**、**串列**、**等號查詢**共用。
_Avoid_：把 `=` 當字面分隔符、離開參考字單獨寫 `=`

**位置錨定**：
同一符號（**slot 連接符** `+`、**等號（韻／聲）** `=`、粵拼字母片段）喺查詢中**唔同字位**可以有唔同意義，係創作者最易混淆嘅根源；**加號錨**、**韻／聲錨**、**粵拼錨**、**串列韻／聲錨**皆屬此族。典型易混：`23o`（兩個字，末字同韻母 o）與 `23+o`（三個字，第 3 個字同韻母 o）。
_Avoid_：把位置錨定當實作術語寫入創作者解釋、用「格」「槽」描述字位（解釋層改用**第 N 個字**）

**通配符**：
`?`、`_`、`%` 三者等價，表示該格**字面不限**。用於缺字各格及韻／聲錨、粵拼錨前後格；只約束字面，唔決定讀音。
_Avoid_：把參考字或粵拼錨當通配符

**音節槽**：
詞長由左至右掃描決定。**0243 碼**每位一音節；**通配符**佔一槽；**slot 連接符** `+` 表多一槽（句首 `+` 例外見 **slot 連接符**）。例：`就=` → 1 槽；`?+就=` → 2 槽。**查詢語意解釋**面向創作者時改說**第 N 個字**，唔直接講槽。
_Avoid_：用串長代替掃描、混淆 `?就=` 與 `?+就=`、把音節槽寫入創作者解釋文案

**slot 連接符**：
規範形 `+`；半形 `*`、全形 `＊`／`﹡`、全形 `＋` 為 **silent 輸入別名**（normalize → `+`）。右邊必須緊接**參考字**或**粵拼錨**，唔可接 0243 碼。禁**相鄰**多個（用 `?` 表通配槽）。句首 `+{字面}…` 為頭格連接，唔多一音節；`X…` 與 `+X…` normalize 等價。Legacy `&`／`·` 碼尾仍映射 `+`。
_Avoid_：把 **查詢分頁** 開關掣 `+` 當查詢運算符、用 `++` 代替 `?`、`+` 後接碼（如 `2+好+3`）、星號錨（舊稱）

**查詢分派優先序**：
領域規則；**缺字型查詢執行**唔得重排。概略順序：
1. **近反義關係查詢**（`~` `!` `~~` `!!` 連接詞複合…）
2. slot 連接符／語法 hint（連續 `+`／`++`、`+` 後接碼…）
3. 前綴通配等號 hint（`?{≥2字}` 漏尾 `=`）、純漢字多錨 hint
4. **前綴通配等號查詢**（`?香港=`）
5. **串列韻／聲錨查詢**（見 ADR-0014；`2=我3` 等碼夾形唔入串列）
6. **等號查詢**、**碼夾等號查詢**
7. **缺字家族**（normalize `+` → **加號錨**、**韻／聲錨**、**通配碼錨**、**粵拼錨**、**碼字查詢**…）
8. **粵拼查詢**、純字面 **詞條 lookup**
_Avoid_：在 executor 重排、把分派細節寫成實作 spec

### 查詢語法家族

**缺字查詢**：
以 0243 碼、**通配符**或漢字描述字面形狀。**首格字面**：`X…` ≡ `+X…`（規範 `+`；`*X…` 仍接受）。**中格字面**：僅漢字**左為通配**時 `?{字}?` ≡ `?+{字}?`；左為碼則走**通配碼錨**（如 `?30人`）。**僅碼＋通配**（`23?`）唔加 `+`。**首格通配＋固定字面**：`?{恰好2字}` 無尾 `=`（如 `?得快`）→ 首格通配、其餘兩格逐字固定，屬缺字查詢；要韻母模板走**前綴通配等號查詢**（`?得快=`）。
_Avoid_：mask 查詢、對 `?30人` 喺參考字前插 `+`、星號錨（舊稱）、把 `?得快` 當前綴通配等號 hint

**碼字查詢**：
連續 0243 碼 + 可選尾**參考字**（如 `23就`、`20人`）；碼位數 = 音節數。無前導 `?`；有前導 `?` 走**通配碼錨**。碼後 `+` 走**加號錨**。
_Avoid_：hybrid（作領域正名）、假設 `?30人` 與 `20人` 同詞長

**字面參考**：
`@` 連接參考字；該格字面須完全一致、唔做同韻擴展（如 `23@就`）。
_Avoid_：@錨

**通配碼錨查詢**：
`{通配}{連續碼}{參考字}`（如 `?30人`）；尾字左為碼 → **參考字**同韻。詞長 = 1 + 碼位數。擴槽：`?30+人`。中間夾 `?` 左至右掃描。
_Avoid_：用串長推詞長、把尾參考字當額外字面格

**韻／聲錨**：
缺字 + **等號（韻／聲）**；詞長 1／2／3 皆屬同一家族。**首格**：`香=?` ≡ `+香=?`。**單字同韻**：`就=`；`?就=` normalize → `就=`（冗餘 `?`，唔加 `+`）。**單字同聲**：`=就`；`?=就` normalize → `=就`。**二字尾格同韻**：`?+就=`。**二字尾格同聲**：`?+=就`。**中格同韻（三字）**：`?{字}=?` normalize → `?+{字}=?`（如 `?港=?` → `?+港=?`）；兩形 **等價、同結果**（normalize 係 silent 改寫，唔報錯）。中間有 **數字**（如 `?3人=?`）屬**串列韻／聲錨**，唔插 `+`。**四字部分韻錨**：`{四字，含 ? 通配，尾 =}`（如 `窮?潦倒=`、`窮困?倒=`、`窮困潦=?`）；`?` 槽任意，其餘格逐格同參考字韻母；**唔**要求字面逐字相等。**四字部分聲錨**：`={四字，含 ? 通配}`（如 `=窮?潦倒`、`=窮困?倒`、`=窮困潦?`）；`?` 槽任意，其餘格逐格同參考字聲母；**唔**要求字面逐字相等。`?{≥2字}=` 且首格為 `?` 者屬**前綴通配等號查詢**；`?={≥2字}` 且首格為 `?` 者屬**前綴通配聲錨查詢**，唔入此二族。
_Avoid_：把 `就=` 當 `?+就=`、把 `=就` 當 `?+=就`、把 `?港=?` 當規範形（規範形係 normalize 後 `?+港=?`）、混淆 `?yut?` 與 `?syut?`、星號錨（舊稱）

**加號錨**：
**位置錨定**子族：**slot 連接符**連接碼與**參考字**或讀音錨。碼尾：`23+就`、`23+就=`；中格：`2+好3`、`2+好=3`；頭格：`+門0`、`+香?30人`；通配碼錨擴槽：`?30+人`。輸入 `*` 仍接受（等同 `+`）。與**粵拼錨**碼尾易混對照見 **位置錨定**。
_Avoid_：把 `?30人` 與 `30+人` 同詞長、把 `+門=0` 同 `+門0` 等價、星號錨（作正名）

**串列韻／聲錨查詢**：
左至右掃描；連續數字每位一音節碼；`?` 僅作單槽通配。每格 **`=` 在碼右**；參考字在 `=` 左 → 韻 `{碼}{字}=`；在右 → 聲 `{碼}={字}`。例：`4困=`（1 字）、`04困=`（2 字）、`04困=49倒=`（4 字韻）、`04=困49=倒`（4 字聲）、`?3人=?`（3 字中格韻）、`23就=`（2 字尾韻）。≥2 參考字走串列；單參考字夾碼仍走**碼夾等號**；含 `+`／`@` 等走原分派。純漢字多錨 → hint（附例 `04困=49倒=`）。`?{碼}{字}?` 無 `=` → hint。詳見 ADR-0014。**唔**等同 `{左碼}{全詞}=`（整詞等號）：串列只約束**有錨格**韻／聲，其餘格僅受 code 數字約束。
_Avoid_：舊 `{碼段}{字}=` 三字讀法、混韻聲於同一串、純漢字 `{字}{字}=`、假設 `04困=49倒=` 與 `0449窮困潦倒=` 同結果

**前綴通配等號查詢**：
`?{參考字≥2}=` → 首音節**完全通配**（聲母、韻母、0243 碼皆不限）+ 自第 2 音節起與**後綴對齊讀音**逐格同韻（唔走整詞 tuple 精確相等）。**唔要求**候選詞字面含參考字或字位逐字相等——第 2 音節起凡符合後綴韻母模板者皆入選（如 `?得快=` 含彭德懷類）。**參考字**韻母取自**後綴對齊讀音**。`?{≥3字}` 漏尾 `=` → hint（`?{恰好2字}` 無 `=` 走**缺字查詢**）。唔收 `?2我=3`；要碼用串列或 `{碼}{參考字}=`。**查詢語意解釋**先述首個字**任意字**，再述第 2 字起至詞尾逐字同參考詞讀音模板同韻，句尾附**押韻標註**（如 `?香港=` → 首個字任意；第 2、第 3 個字同「香港」同韻（雙押））。
_Avoid_：與 `{字}=` 單字韻／聲錨混、與碼夾等號混、把前綴通配當整詞等號查詢、要求字面含參考字子串、解釋寫成整詞同韻而唔標首字任意

**後綴對齊讀音**：
**前綴通配等號查詢**所用**參考字**韻母／聲母列：在已收錄詞中找字面**以參考字結尾**者，取權威讀音後截取對應後綴音節列。多候選時**先取字面長度大於參考字長度**（成語／長詞後綴）；無則取與參考字等長之獨立詞；同層再依 **pron_rank** → **Essay 詞頻** → 粵拼序（同**等號參考讀音**平手規則）。無任何後綴對齊候選時，退回**等號參考讀音**（整段參考字詞條單列）。
_Avoid_：後綴對齊（作實作名）、一律用獨立短詞讀音當成語後綴模板、長短詞混排只按詞頻

**等號查詢**：
整詞逐音節同韻或同聲。`詞=` 比韻；`=詞` 比聲。參考字至少二字。`{左碼}{詞}=`（左碼位數=詞長，如 `0449窮困潦倒=`）→ 整詞同韻 **且** 候選詞 0243 碼須與左碼等長且匹配（**0243模式**含等價變體）；**所有**符合者皆入選（唔限參考字面）。**參考讀音**：**四字**字面（成語）且左碼與詞長一致時，若參考詞有 code 落入左碼變體嘅詞條則優先用該條韻母／聲母列，否則取**等號參考讀音**；**二字及以下**一律取**等號參考讀音**（左碼只約束候選，唔用左碼揀參考列——短詞多讀音標音易錯）。**唔**等同 `{詞}=`（純整詞同韻、無左碼約束）。**唔**等同**串列韻／聲錨**。零結果時：參考詞**有收錄**但左碼下無整詞同韻候選 → hint；參考詞本身搵唔到 → 靜默空。
_Avoid_：單字整詞等號（改用韻／聲錨）、當純字面詞條 lookup、當串列韻錨、只返參考字面一條、假設 `{左碼}{詞}=` 與 `{詞}=` 或串列同結果、零結果一律 hint、二字詞用左碼揀參考讀音列、把左碼當參考字收錄碼 substring

**碼夾等號查詢**：
`{左碼}{=?}{參考字}{=?}{右碼}`（如 `2=我3`、`34=我`）。須同時滿足整詞 0243 碼與錨位一維（聲或韻）；唔要求該格字面係參考字。
_Avoid_：用參考字整段讀音代替碼位約束、多讀音 OR

**粵拼錨**：
**缺字型查詢家族**內以拉丁粵拼片段取代**參考字**；與**粵拼查詢**（整段查詞）無關。分三子類（下）。片段無效 → 拒絕並提示。**近反義關係查詢**唔收。
_Avoid_：拉丁錨（作產品名）、把粵拼查詢同粵拼錨混談

**聲母粵拼錨**：
單輔音或叢集 `{首碼}{片段}{末碼}`（如 `3h4`、`3gw4`、`2ng0`）。`{碼}m{碼}`／`?m?` 等可能歧義 → **雙列結果**（聲母列 vs 獨立鼻音韻母列）；`ngo` 的 `ng` ≠ 獨立韻母 `ng`。
_Avoid_：假設片段必為聲母或必為韻母

**韻母粵拼錨**：
韻母片段或獨立 `m`／`ng`（等價）。例：`23o`（二字末格韻，≡ `23我=`）、`?yut?`、`?m?`。`?u?` 與 `?yu?` **唔**互通（見 **u／yu 韻部**）。二字 `23o` **唔**等同三字 `23+o`／`23+我=`。
_Avoid_：把 `23o` 同 `23+o` 合併、用 `?u?` 當 `?yu?`

**完整音節粵拼錨**：
該格粵拼（無調）完全一致。二字首格：`3hon4`（碼 `34`）；三字中格：`3?hon4`；二字末格：`23ngo`（≠ `23o` 韻母錨）。
_Avoid_：把 `3?hon4` 同 `3hon4` 混詞長

**漢字完整音節錨**：
`$` 後**僅**接單一漢字；該格約束為該字 rime **預設讀音**嘅完整音節字母（唔含聲調），語意等同對應**完整音節粵拼錨**（如 `3$漢4` ≡ `3hon4`）。**字位範圍**鏡像今日所有**完整音節粵拼錨**位（首格、末格、三字中格等）。全形 `＄` normalize 為 `$`。輸入時 **normalize** 為拉丁音節錨再走現有分派（唔新增查詢種類）。多音字唔展開；預設讀音無效 → 拒絕並提示。
_Avoid_：`$` 後接拉丁粵拼、用非預設讀音展開 OR、把 `$` 當 slot 連接符、只支援首格而唔鏡像其他完整音節錨位

**同音節疊字查詢**：
以 `$$` 為核心標記（鏡像 **近義複合** `~~` 語法骨架）：`$$`、`{碼}$$`、`$${字}`、`{碼}$${字}`；搵**二字詞**且兩字音節字母相同（聲調唔限）。比對用該詞**收錄粵拼**逐音節；**任一讀音列**內第 1、2 音節字母相同即入選。結果**按字面合併**，展示該詞所有符合條件嘅讀音列。候選來自 **build 快照**（同 **近義複合** `~~`）；查詢時再套碼前綴與尾錨縮窄。全形 `＄＄` normalize 為 `$$`。碼前綴受 **0243搜尋模式** m1/m2 等價約束；尾字錨縮窄同 `~~` 尾錨規則。**近反義關係查詢**唔收。
_Avoid_：逐字 rime 預設代替詞級標音、要求所有讀音列同時滿足、把 `$$` 當漢字完整音節錨、與 `~~` 近義語意混淆

**同音異讀查詢**：
以 `{左碼}/{右碼}` 為標記；搵**同一字面**且詞庫內存在**至少兩個唔同讀音列**，分別符合左、右碼位模板（m1/m2 **0243搜尋模式**等價）。左右**等長**；已寫數字約束該碼位，未寫位以 **`?`** 通配（如 `?3/?4` 只約束第 2 字碼；`33/34` 二字四位全寫死）。詞長 = 模板長度；不等長 → 拒絕並提示。入選須**至少一列符合左、至少一列符合右**，且參與嘅左右兩列**粵拼唔同**。結果**按字面合併**，每條讀音標註符合**左**／**右**／**左·右**。候選來自**啟動預載記憶體索引**（`char` → 多讀音列；只含 `COUNT(DISTINCT jyutping) > 1` 嘅字面；**唔**增大 `lyrics.db`）；查詢時做碼位模板過濾。**僅 0243搜尋模式**；整段須符合 `^[\d?]+/[\d?]+$`（全形 `／` normalize 為 `/`）；含漢字或其他符號 → 拒絕並提示。**近反義關係查詢**唔收。
_Avoid_：用 `_` 或其他通配符、左右唔等長、單一讀音或同一粵拼重複入選、扁平重複字面、每次查詢裸掃全表、同 **詞條 lookup** 混談

**粵拼查詢**：
整段粵拼查詞（如 `syut`、`ming4 baak6`）。無調逐音節比對（忽略調）；全調須完全一致；混合按音節分級。**近反義模式**唔收。
_Avoid_：jyutping fragment（作領域正名）

**詞條 lookup**：
純漢字精確查詢（如「香港」「開心」）；回傳讀音、0243 碼及**詞條 lookup 版面**編排。
_Avoid_：WordLookup（作領域正名）

## 詞庫與排序

**詞庫**：
決定哪些字面是「真詞」、以及詞級讀音（字面＋粵拼＋0243 碼）的權威來源。搜尋用的詞條庫是詞庫的副本；注入須經**收錄決策**（單字 rime、多字詞級標音、或音節拼接讀音），禁止憑空造字或繞過收錄決策猜音。
_Avoid_：把自動猜讀當詞庫、測試用造字進正式收錄

**詞庫埠**：
詞庫資料的抽象邊界：**raw lookup**（rime 單字表、詞級標音 JSON，不帶收錄判斷）與**排序信號**（essay 詞頻、curated、pron_rank）。**收錄門檻**（能否注入、讀音歸哪一類）由**收錄決策**單獨判定，不由埠的 lookup 方法隱含決定。
_Avoid_：把 runtime 詞條庫本身稱作詞庫、把收錄決策塞進 raw lookup 回傳值

**詞庫快取索引**：
詞條庫在 runtime 記憶體中的可查索引與就緒狀態；就緒後供搜尋路徑依字面形狀與聲韻維度取向量候選，未就緒時搜尋降級直查詞條庫。**離線啟動預載**的編排與磁碟暖啟屬餵入此索引的實作細節，不與索引介面混稱。**免安裝交付**的 Portable 套件可內含與捆綁 `lyrics.db` 對應的預暖磁碟快照，縮短首次**就緒閘**等待。
_Avoid_：word_cache（作領域正名）、把預載執行緒或磁碟快照當成索引定義的一部分

**多字收錄**：
長度 ≥ 2 的字面，僅當詞庫有該詞的**詞級標音**（粵拼＋0243 碼）時，才以詞級標音**收錄**進詞條庫；可有多個讀音列。詞級標音未命中時，不由此路徑注入（改走**音節拼接讀音**收錄路徑，若拼接成功）。
_Avoid_：逐字拼接多音字讀音後冒充詞級標音、把拼接結果稱作詞級標音

**詞級異讀**：
同一**字面**、多條相異**粵拼**嘅**詞條**（如「今晚」`gam1 maan5` 與 `gam1 maan1`）。**合理**與否以**詞級標音上游**列出為準（口語／書面、多調等詞級現象）；**rime 單字表**唔做異讀裁判，亦唔用音節拼接預設否定上游異讀。**詞條從源重建** 合併 **詞級標音** 時**唔**以 rime 音節內容過濾讀音。上游日後列出罕見或錯誤讀音時，照收錄，由 **詞庫勘誤** overlay 人手刪改，唔靠自動 rime 過濾代替勘誤。
_Avoid_：用 rime 預設拼接否定詞級異讀、用 pron_rank 擋多字讀音、build 合併時用 rime 音節存在性過濾詞級標音、把異讀與上游錯誤混為「自動過濾」處理

**詞級讀音合併**：
**詞條從源重建** 合併 **詞級標音** 時，以 **(字面, 粵拼)** 去重；同一字面之相異粵拼須保留為多條**詞條**（支援**詞級異讀**與**異讀碼查詢**）。**同層**（同一上游 ingest batch）：相異粵拼**全收**。**跨層**：高優先來源已出現過嘅字面，低優先來源**只可**合併**相同**粵拼嘅來源紀錄，**唔可**新增粵拼；低優先仍可補**前序完全無**嘅新字面（含該字面嘅全部讀音列）。進入合併前須通過 **詞級標音格式門檻**。
_Avoid_：二字字面只留第一條粵拼、跨層低優先加新粵拼、把 merge 當單讀音覆蓋、收語法損壞嘅粵拼

**詞級標音格式門檻**：
**詞級標音** ingest／合併嘅信任邊界校驗：粵拼 token 語法合法、音節數與**字面**長度一致、可衍生 **0243 碼**；唔合格行**靜默丟棄**（可記 log／計數），**唔** fail 成個 **詞條庫建置命令**。**唔**查 rime 該字有冇該音節（語義合理性交 **詞級標音上游** 與 **詞庫勘誤**）。
_Avoid_：用 rime 音節存在性當格式門檻、壞行令 build fail fast、把格式門檻當異讀過濾

**詞條庫建置命令**：
maintainer 單一入口（如 `python -m ingest build-db`）執行 **詞條從源重建** 全條龍：truncate 查詢核心表 → lexicon ingest → **詞庫勘誤** overlay → 近反義 `normalize` → `build-relations` → `ingest-cilin` → `expand-antonyms-cilin` → `expand-antonyms-mirror` → `ingest-compound-ant` → **近義橋反義快照** inject → **關係補錄清單** 套用 → export 與 README 詞條數；可選 copy 至 PWA `public/`。**唔**預設跑 embedding 橋接烘焙。`client` prebuild 僅 copy 已建 **詞條庫**。**詞條庫建置互斥**：同一 **詞條庫** 同時只准一個建置進程；並行會因 SQLite 單寫者嚴重拖慢。
_Avoid_：把 client prebuild 當 SSOT compile、創作者 build 依賴 raw 字典、預設 build 跑 embedding 橋接、跳過 relations 只 rebuild words、無鎖並行多個 build-db

**詞條從源重建**：
maintainer 由 **詞條 SSOT 來源** manifest 逐源 ingest、合併、套用 **詞庫勘誤** overlay，**全量替換** **詞條庫** 內 `words`（及 **詞條來源紀錄**）與 `word_relations` 等查詢核心表，產出下游產物（export json、README 詞條數、PWA copy 等）；**唔**以現行 **詞條庫** 作輸入權威。每次 build 全量重算，唔依賴 `pending`／`applied` 狀態機；**唔**保留 runtime **收錄決策** 注入列。
_Avoid_：把 copy 現行 **詞條庫** 叫從源重建、增量 patch 當權威、rebuild 時 merge runtime 注入列、跳過 overlay 直接改 raw 檔

**詞條 SSOT 來源**：
重建 **詞條庫** 時嘅**唯一權威輸入**：maintainer fetch 嘅**未修改原始字典檔**——預設 **rime** 單字表、**words.hk**、**開放詞典**、專案 **Curated 常用詞**（**詞級標音 curated**）；以 **詞條源清單** 列舉，每源獨立 ingest。**唔**含 **CC-Canto**（見 **CC-Canto 脫離**）。**唔**以 legacy **詞條庫**、`data/raw/clean` 混合 JSON 或 runtime **音節拼接讀音** 作 build 輸入。
_Avoid_：把現行 **詞條庫** 當 SSOT、把拼接結果寫入 build、未列清單嘅檔案混入 ingest、把 CC-Canto 當預設源

**詞條源清單**：
`data/lexicon/sources.yaml` 列舉 **詞條 SSOT 來源** 與 ingest 模組；`enabled`（或註解該項）決定該次 build 是否納入。停用源唔需要 raw 檔、唔參與合併；**啟用**源缺 raw 則 **詞條庫建置命令** fail fast。增刪詞典 ≈ 改清單一行 + **詞條從源重建**（wipe → reingest → overlay）；**唔**版控完整 **詞條庫** 本體（`lyrics.db` gitignore）。
_Avoid_：手改 **詞條庫** 當維護方式、停用源仍要求 raw 存在、增源要改多個散落腳本、把完整 **詞條庫** commit 入 git

**詞條庫 fixture**：
CI 與煙霧測試用嘅細 **詞條庫** 快照（如 `tests/fixtures/lyrics.db`），版控追蹤；完整 **詞條庫** 由 **詞條庫建置命令** 本機產出，Release 上傳產物。
_Avoid_：用 fixture 當 Release 資產、CI 每次 PR 跑完整 build-db、把 fixture 當 SSOT

**CC-Canto 脫離**：
**詞條從源重建** 與 **詞條源清單** **唔**再納入 CC-Canto（結案 GitHub issue #9 方向）。既有 **詞條庫** 內 CC 來源讀音須經 wipe reingest 清除，唔靠增量刪除。
_Avoid_：預設 manifest 仍含 CC-Canto、為過渡保留 CC merge 分支、Release 仍標 CC BY-SA 混合（脫離後）

**詞級標音上游**：
**詞條 SSOT 來源** 入面、多字 **詞級標音** 嘅認可第三方：**words.hk**（公有領域）、**開放詞典 · 粵語詞典**（CC BY 3.0）。合併**優先序**：**詞庫勘誤** → words.hk → 開放詞典（僅補 words.hk 無之字面）→ **詞級標音 curated**（僅補前序仍無之字面）；單字讀音由 rime 單字表 SSOT 提供。**音節拼接讀音**只作 runtime **收錄決策** fallback，**唔**寫入 build 或副件 export。跨層讀音邊界見 **詞級讀音合併**（低優先唔得為已 claim 字面加新粵拼）。
_Avoid_：未列上游來源混入 export、低優先為高優先已有字面加異讀、開放詞典與 words.hk 同字面無 tie-break 全收、把拼接結果 export 進副件

**音節拼接讀音**：
詞庫無該多字詞級標音時，逐字取 rime 單字「預設」讀音拼成整詞粵拼與 0243 碼；用於查詢、結果頁顯示與參考字錨點解析。拼接成功時可作**收錄決策**的第三條路徑寫入詞條庫，但**不**宣稱為詞級標音、**不**與詞級標音混稱。任一字缺 rime 預設則拼接失敗；單字有多個 rime 讀音時**只取預設**，不展開讀音組合。
_Avoid_：與詞級標音混稱、把拼接結果當詞庫權威、線上 API 猜音

**單字收錄**：
長度 1 的字面，詞條庫已有則不重複注入；否則從 rime 單字表取「預設」讀音注入（runtime **收錄決策** fallback）。**詞條從源重建** 時 rime SSOT ingest **包含** `char.csv` 全部有效讀音列（預設、常用、罕見），唔限預設。
_Avoid_：為排序而單獨收錄 essay 裡出現但未收錄的字、build 只 ingest rime 預設讀音、把 fallback 預設規則當 build 範圍

**Essay 語料**：
粵語書面／口語詞頻語料（[rime-cantonese](https://github.com/rime/rime-cantonese) · [essay-cantonese.txt](https://github.com/rime/rime-cantonese/blob/main/essay-cantonese.txt)，CC BY 4.0），格式為「詞＋詞頻」。**不**開注入門、**不**用於猜讀音；僅供排序。語料隨 repo 提供。
_Avoid_：把語料當詞庫、把語料當近義來源

**Essay 詞頻索引**：
啟動時自 Essay 語料建立的**字面→詞頻**查表，僅供搜尋排序；載入記憶體，不寫入詞條庫、不另建詞頻表。更新語料檔後重啟即生效。
_Avoid_：詞頻表（作資料庫實體）、把詞頻存進詞條列、用 vector embedding 取代詞頻排序

**Curated 常用詞**：
專案維護的填詞常用字面列表，在 essay 詞頻相同時作排序加成。
_Avoid_：把 curated 排在 essay 之前（現行 tier 為 essay 優先）

**pron_rank**：
rime 單字表的讀音常用度標記（預設＞常用＞罕見），用於單字讀音選擇與排序 tie-break；多字讀音以詞級標音為準。
_Avoid_：把 pron_rank 當收錄門檻

**詞條排序信號**：
**搜尋結果排序**與**等號參考讀音選列**共用的可比較信號定義（純漢字、Essay 詞頻、curated、pron_rank、字面等）；兩條契約 tier 順序不同，但信號來源一致，由同一模組對外提供兩條 key。
_Avoid_：把兩條契約收成一條 tier、essay_sort（實作名）

**等號參考讀音選列**：
多讀音字面在**等號參考讀音**情境下選權威單列的 tie-break：**pron_rank → Essay 詞頻 → 略過 aa 變體 → 粵拼序**；信號來自**詞條排序信號**，tier 順序與**搜尋結果排序**不同。
_Avoid_：與搜尋結果排序共用同一 tier 順序、把選列規則散落各呼叫端

**詞條 lookup 版面**：
純漢字精確查詢（詞條 lookup）回傳的多段結果編排：code header → 粵拼 header → 查詢詞本身 → 同碼同韻段（共用字字優先）→ 尾字韻錨段 → 同碼餘下；各詞條段內套用**搜尋結果排序**。快取暖化不藏在版面模組內，由呼叫端明示執行。
_Avoid_：code_aware_ranker（作領域正名）、在版面模組內隱式 sync 詞庫快取索引

**缺字查詢字面優先**：
**缺字查詢**結果在同一匹配層級內，先比**參考字**位置的字面吻合數（愈多愈前），再套用**搜尋結果排序**；不得在字面優先 tier 內另訂 essay／curated 順序。
_Avoid_：mask_priority（作領域正名）、把字面吻合與 essay 詞頻混為同一維度

**搜尋結果排序**：
扁平詞條清單在同一匹配層級內，依**詞條排序信號**套用 tier：**純漢字字面 → essay 詞頻 → curated → pron_rank → 字面**。**缺字查詢字面優先**、code header、近反義分組等結構性區塊順序不變，只在區塊內套用上述信號。目標是貼近 0243 常用度體感，不要求與 0243.hk 逐詞一致。
_Avoid_：預設字母序、每種語法各一套 tie-break、混雜粵拼字面與純漢字無區分、維護 0243 完整快照作 CI 門檻
